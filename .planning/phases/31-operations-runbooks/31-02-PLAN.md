  <name>Task 1: Create troubleshooting guide</name>
  <files>TROUBLESHOOTING.md</files>
  <action>Create TROUBLESHOOTING.md at the project root covering common issues and solutions:

  **Structure — organized by symptom:**

  1. **Build & Startup Issues**
     - "Docker build fails" — check .dockerignore, node version, missing build args
     - "Server won't start" — check env vars (run envValidator), port conflicts, missing dependencies
     - "Frontend build fails" — check Clerk keys format, NEXT_PUBLIC_API_URL, Node version
     - "npm install fails" — clear node_modules and lockfile, check Node version

  2. **Authentication Issues**
     - "401 on all API requests" — check Supabase JWT, Bearer token format, CORS
     - "Clerk login not working" — check NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY, domain config
     - "CORS errors in browser" — check ALLOWED_ORIGINS env var, verify origin match

  3. **Integration Issues**
     - "Claude AI analysis returns errors" — check ANTHROPIC_API_KEY, rate limits, model availability
     - "Plaid connection fails" — check PLAID_CLIENT_ID/SECRET, environment (sandbox/dev/prod)
     - "Supabase connection refused" — check SUPABASE_URL, SUPABASE_SERVICE_KEY, RLS policies
     - "Redis connection fails" — check REDIS_URL, verify Redis is running

  4. **Runtime Issues**
     - "High memory usage" — check /metrics, restart container, review for memory leaks
     - "Slow response times" — check /metrics p95/p99, database queries, external API latency
     - "File upload fails" — check file size limits, allowed MIME types, storage configuration

  5. **Development Environment**
     - "Tests fail locally" — check .env file, ensure correct Node version, npm ci
     - "Docker Compose services won't connect" — check service names, network, depends_on
     - "Hot reload not working" — check nodemon config, volume mounts in Docker

  Format each entry as:
  ```
  ### Problem: [symptom]
  **Cause:** [likely cause]
  **Fix:** [numbered steps]
  ```

  Keep solutions to 3-5 steps max. Reference ENV-GUIDE.md for env var details.</action>
  <verify>TROUBLESHOOTING.md exists, covers at least 15 common issues, organized by category</verify>
  <done>Troubleshooting guide covers build, auth, integration, runtime, and dev environment issues</done>
</task>
