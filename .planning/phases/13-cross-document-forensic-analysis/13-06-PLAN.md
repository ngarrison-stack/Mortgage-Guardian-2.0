<task type="auto">
  <name>Task 1: Create forensic analysis API routes and validation schemas</name>
  <files>backend-express/routes/cases.js, backend-express/schemas/cases.js, backend-express/server.js</files>
  <action>
Add two new endpoints to the cases router for cross-document forensic analysis. These MUST be placed BEFORE the `/:caseId` route in the router to prevent Express treating "forensic-analysis" as a caseId parameter (same pattern as 12-03 with analysis route).

**New endpoints:**

**POST /v1/cases/:caseId/forensic-analysis** — Trigger cross-document forensic analysis
- Auth: requireAuth middleware (userId from req.user.id)
- Validation: Joi schema for params (caseId) + optional body:
  ```
  body: {
    plaidAccessToken: Joi.string().optional(),
    transactionStartDate: Joi.string().pattern(/^\d{4}-\d{2}-\d{2}$/).optional(),
    transactionEndDate: Joi.string().pattern(/^\d{4}-\d{2}-\d{2}$/).optional(),
    dateTolerance: Joi.number().integer().min(0).max(30).optional(),
    amountTolerance: Joi.number().min(0).max(100).optional()
  }
  ```
- Handler logic:
  1. Extract caseId from params, userId from req.user.id
  2. Build options from body (plaidAccessToken, date range, tolerances)
  3. Call forensicAnalysisService.analyzeCaseForensics(caseId, userId, options)
  4. If result.error: return 200 with { caseId, status: 'error', error: result.errorMessage }
  5. If success: return 200 with { caseId, status: 'completed', forensicAnalysis: result }
- Error responses:
  - 400: Validation failure (Joi)
  - 401: Unauthorized (middleware)
  - 200 + status:'error': Analysis-level failures (case not found, insufficient docs)
  - 500: Unexpected server errors

**GET /v1/cases/:caseId/forensic-analysis** — Retrieve stored forensic analysis
- Auth: requireAuth middleware
- Validation: Joi schema for params (caseId)
- Handler logic:
  1. Query Supabase for case's stored forensic analysis (from case_files table or wherever 13-05 persists it)
  2. Verify case belongs to userId (user isolation)
  3. If found: return 200 with { caseId, status: 'completed', forensicAnalysis: stored result }
  4. If not found: return 404 with { error: 'NotFound', message: 'No forensic analysis found for this case' }

**Schemas to add in schemas/cases.js:**
- forensicAnalysisParamsSchema: { caseId: Joi.string().required() }
- forensicAnalysisBodySchema: body validation as described above

**Route placement in cases.js:**
Place the forensic-analysis routes BEFORE the generic /:caseId routes:
```javascript
// Forensic analysis routes (MUST be before /:caseId)
router.post('/:caseId/forensic-analysis', validate(forensicAnalysisBodySchema), async (req, res) => { ... });
router.get('/:caseId/forensic-analysis', async (req, res) => { ... });

// Existing routes
router.get('/:caseId', async (req, res) => { ... });
```

**Update server.js:**
Update the 404 handler's available routes list to include the new forensic analysis endpoints.

Use createLogger('forensic-analysis-routes') for structured logging within the route handlers.
