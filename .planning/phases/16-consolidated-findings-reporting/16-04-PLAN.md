<task type="auto">
  <name>Task 1: Create RESPA dispute letter service</name>
  <files>backend-express/services/disputeLetterService.js</files>
  <action>
Create a singleton service for generating RESPA-compliant dispute letters using Claude AI.

Methods:
1. `generateDisputeLetter(letterType, consolidatedReport, options = {})` — main entry point
   - letterType: 'qualified_written_request' | 'notice_of_error' | 'request_for_information'
   - Validates letterType against LETTER_TYPES from config
   - Builds a domain-specific prompt with:
     a. System prompt: "You are a mortgage servicing law expert. Generate a formal RESPA-compliant {letterType}..."
     b. User prompt: includes borrower info, servicer info, specific violations/findings with statutory citations, requested remedies
   - Calls Claude API (claude-sonnet-4-5, temperature 0.1) requesting JSON response with:
     - `subject` (string) — letter subject line
     - `salutation` (string)
     - `body` (string) — full letter body in markdown
     - `demands` (string[]) — specific demands/requests
     - `legalCitations` (string[]) — RESPA/TILA/etc. citations referenced
     - `responseDeadline` (string) — e.g., "30 days" per RESPA Section 6
     - `closingStatement` (string)
   - Returns: { letterType, generatedAt, content: { subject, salutation, body, demands, legalCitations, responseDeadline, closingStatement }, recipientInfo: { servicerName, servicerAddress } }
   - On Claude API failure: returns { error: true, errorMessage, letterType } — never throws

2. `_buildLetterPrompt(letterType, report)` — constructs the Claude prompt
   - For qualified_written_request: focuses on specific errors with RESPA Section 6 requirements (30-day acknowledgment, 60-day response)
   - For notice_of_error: focuses on identified errors per 12 CFR 1024.35 with specific error categories
   - For request_for_information: focuses on requesting documents/records per 12 CFR 1024.36
   - Include violation summaries, document references, and severity levels in the prompt

3. `_extractRecipientInfo(report)` — extracts servicer info from case summary

Follow the exact pattern from complianceAnalysisService.js:
- Lazy Anthropic client init (`let anthropicClient = null; function getClient() { ... }`)
- Markdown code fence JSON extraction fallback
- Structured logging with createLogger('dispute-letter')
