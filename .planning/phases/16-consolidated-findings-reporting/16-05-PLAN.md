<task type="auto">
  <name>Task 1: Create consolidated report orchestrator service</name>
  <files>backend-express/services/consolidatedReportService.js</files>
  <action>
Create a singleton orchestrator service that coordinates the full consolidated report generation flow:

**Pipeline steps:**
1. **GATHER** — call reportAggregationService.gatherCaseFindings(caseId, userId) to collect all upstream data
2. **SCORE** — call confidenceScoringService.calculateConfidence(aggregatedData) for confidence scores + determineRiskLevel()
3. **LINK** — call confidenceScoringService.buildEvidenceLinks(aggregatedData) to connect findings to source docs
4. **GENERATE RECOMMENDATIONS** — build prioritized recommendations from findings using RECOMMENDATION_PRIORITY config
5. **DISPUTE LETTER** (optional) — if options.generateLetter, call disputeLetterService.generateDisputeLetter(letterType, report)
6. **ASSEMBLE** — build final report matching consolidatedReportSchema
7. **VALIDATE** — run validateConsolidatedReport(), attach warnings
8. **PERSIST** — best-effort save to Supabase consolidated_reports table

Method: `generateReport(caseId, userId, options = {})`
- options.generateLetter (boolean, default false)
- options.letterType (string, default 'qualified_written_request')
- options.skipPersistence (boolean, default false)

Returns: consolidated report object or { error: true, errorMessage }

**Design principles (matching existing orchestrators):**
- Each step wrapped in try/catch with stepMeta timing
- Warnings array collects non-fatal issues
- If Step 1 (GATHER) fails completely → return error (can't proceed without data)
- Steps 2-5: graceful degradation — if scoring fails, use defaults; if letter fails, set disputeLetterAvailable: false
- Generate reportId with uuid or crypto.randomUUID()
- Use createLogger('consolidated-report') for structured logging
- `module.exports = new ConsolidatedReportService()` singleton pattern

**Recommendation generation logic (_generateRecommendations):**
- Iterate all findings (anomalies, discrepancies, violations)
- For each finding: map to action using RECOMMENDATION_MAP-style mapping (similar to forensicAnalysisService RECOMMENDATION_MAP)
- Deduplicate by action text
- Sort by priority (critical findings → priority 1)
- Attach relatedFindingIds and optional legalBasis from violation citations
