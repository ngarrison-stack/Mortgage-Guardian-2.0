<task type="auto">
  <name>Task 1: Create consolidated audit report Joi schema</name>
  <files>backend-express/schemas/consolidatedReportSchema.js</files>
  <action>
Create a Joi schema defining the unified audit report structure. The report aggregates data from three upstream sources:

1. **Individual document analysis** (from analysisReportSchema.js): extractedData, anomalies, completenessScore per document
2. **Cross-document forensic analysis** (from crossDocumentAnalysisSchema.js): discrepancies, timelineViolations, paymentVerification
3. **Federal + state compliance** (from complianceReportSchema.js): violations (federal + state), riskLevel, legalNarrative

Schema structure:
- `reportId` (string, required) — UUID
- `caseId` (string, required)
- `userId` (string, required)
- `generatedAt` (ISO date string, required)
- `reportVersion` (string, default '1.0')
- `caseSummary` — object with borrowerName, propertyAddress, loanNumber, servicerName, documentCount, caseCreatedAt
- `overallRiskLevel` — enum: critical, high, medium, low, clean
- `confidenceScore` — object with overall (0-100), breakdown: { documentAnalysis, forensicAnalysis, complianceAnalysis } each 0-100
- `findingSummary` — object with totalFindings (number), bySeverity: { critical, high, medium, low, info } (counts), byCategory: { documentAnomalies, crossDocDiscrepancies, timelineViolations, paymentIssues, federalViolations, stateViolations } (counts)
- `documentAnalysis` — array of per-document summaries: { documentId, documentName, type, subtype, completenessScore, anomalyCount, keyFindings[] }
- `forensicFindings` — object with discrepancies[], timelineViolations[], paymentVerification (nullable)
- `complianceFindings` — object with federalViolations[], stateViolations[], jurisdiction (nullable)
- `evidenceLinks` — array of { findingId, findingType, sourceDocumentIds[], evidenceDescription, severity }
- `recommendations` — array of { priority (1-5), category, action, legalBasis (nullable), relatedFindingIds[] }
- `disputeLetterAvailable` (boolean, default false)
- `disputeLetter` (nullable object with letterType, generatedAt, content, recipientInfo)
- `metadata` — object with generationDurationMs, stepsCompleted[], warnings[]

Export: `validateConsolidatedReport(report)` helper (returns { valid, errors, warnings } — validation as warnings pattern), `RISK_LEVEL_PRIORITY` map, `FINDING_CATEGORIES` enum.

Follow the exact pattern from complianceReportSchema.js: Joi.object() with labeled schema, .options({ stripUnknown: false }), and a validate helper that returns warnings not rejections.
  </action>
  <verify>node -e "const s = require('./backend-express/schemas/consolidatedReportSchema'); console.log(typeof s.validateConsolidatedReport, Object.keys(s));"</verify>
  <done>Schema exports validateConsolidatedReport, RISK_LEVEL_PRIORITY, FINDING_CATEGORIES. Joi validation passes for well-formed reports, returns warnings (not throws) for minor deviations.</done>
</task>

<task type="auto">
  <name>Task 2: Create confidence scoring weights and evidence linking configuration</name>
  <files>backend-express/config/consolidatedReportConfig.js</files>
  <action>
Create a configuration file defining:

1. **Confidence scoring weights** — how much each analysis layer contributes to overall confidence:
   - SCORING_WEIGHTS: { documentAnalysis: 0.30, forensicAnalysis: 0.35, complianceAnalysis: 0.35 }
   - Per-layer scoring factors:
     - documentAnalysis: completenessWeight (0.4), anomalyPenalty (0.6) — higher completeness = higher confidence, more anomalies = lower confidence
     - forensicAnalysis: discrepancyPenalty (0.5), timelinePenalty (0.3), paymentPenalty (0.2)
     - complianceAnalysis: violationPenalty (0.7), severityMultiplier: { critical: 4, high: 3, medium: 2, low: 1, info: 0 }

2. **Risk level thresholds** — map confidence score to risk level:
   - RISK_THRESHOLDS: { critical: { maxScore: 25 }, high: { maxScore: 50 }, medium: { maxScore: 70 }, low: { maxScore: 90 }, clean: { maxScore: 100 } }

3. **Evidence linking categories** — finding type to evidence mapping:
   - EVIDENCE_CATEGORIES mapping each finding type (anomaly, discrepancy, timelineViolation, paymentIssue, federalViolation, stateViolation) to source description templates

4. **Recommendation priority mapping** — severity to priority:
   - RECOMMENDATION_PRIORITY: { critical: 1, high: 2, medium: 3, low: 4, info: 5 }

5. **RESPA dispute letter config**:
   - LETTER_TYPES: ['qualified_written_request', 'notice_of_error', 'request_for_information']
   - LETTER_SECTIONS: array of required sections per letter type

Export all constants. Follow the exact pattern of config files like crossDocumentComparisons.js — named exports of plain objects with JSDoc comments.
  </action>
  <verify>node -e "const c = require('./backend-express/config/consolidatedReportConfig'); console.log(Object.keys(c)); console.log('weights sum:', c.SCORING_WEIGHTS.documentAnalysis + c.SCORING_WEIGHTS.forensicAnalysis + c.SCORING_WEIGHTS.complianceAnalysis);"</verify>
  <done>Config exports SCORING_WEIGHTS (summing to 1.0), RISK_THRESHOLDS, EVIDENCE_CATEGORIES, RECOMMENDATION_PRIORITY, LETTER_TYPES, LETTER_SECTIONS. All constants are plain objects with JSDoc.</done>
