<feature>
  <name>Report Data Aggregation Service</name>
  <files>backend-express/services/reportAggregationService.js, backend-express/__tests__/services/reportAggregationService.test.js</files>
  <behavior>
    gatherCaseFindings(caseId, userId) → { caseInfo, documentAnalyses, forensicReport, complianceReport, errors }

    Cases:
    - Full data case → returns all 3 sections populated with normalized findings
    - Case with no forensic report → forensicReport: null, errors includes forensic note
    - Case with no compliance report → complianceReport: null, errors includes compliance note
    - Empty case (no documents) → documentAnalyses: [], forensicReport: null, complianceReport: null
    - Invalid caseId → returns { error: true, errorMessage }
    - Partial forensic (some pair failures) → includes available data, errors notes partials

    normalizeDocumentAnalysis(analysisReport) → { documentId, documentName, type, subtype, completenessScore, anomalyCount, anomalies[], keyFindings[] }

    Cases:
    - Complete analysis → all fields populated
    - Missing extractedData → keyFindings: []
    - Missing anomalies → anomalies: [], anomalyCount: 0

    extractFindingSummary(documentAnalyses, forensicReport, complianceReport) → { totalFindings, bySeverity, byCategory }

    Cases:
    - All sources → counts from all categories
    - Null forensic → forensic categories are 0
    - Null compliance → compliance categories are 0
  </behavior>
  <implementation>
    Singleton service that:
    1. Calls caseFileService.getCase(caseId, userId) for case info + document list
    2. For each document, retrieves analysis report from Supabase or pipeline state (fallback)
    3. Retrieves forensic report from Supabase (forensic_analyses table) or returns null
    4. Retrieves compliance report from Supabase (compliance_reports table) or returns null
    5. Normalizes all findings into unified shapes
    6. Never throws — returns error objects on failures, partial data on partial failures
    Mock external boundaries (Supabase, caseFileService) in tests, test normalization logic for real.
  </implementation>
</feature>
