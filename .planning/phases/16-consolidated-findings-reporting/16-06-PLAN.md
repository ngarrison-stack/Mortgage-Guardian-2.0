  <action>
**1. Create Joi request schemas (backend-express/schemas/reports.js):**

- `generateReportSchema` — POST /v1/cases/:caseId/report
  - params: { caseId: Joi.string().required() }
  - body: { generateLetter: Joi.boolean().default(false), letterType: Joi.string().valid('qualified_written_request', 'notice_of_error', 'request_for_information').default('qualified_written_request'), skipPersistence: Joi.boolean().default(false) }

- `getReportSchema` — GET /v1/cases/:caseId/report
  - params: { caseId: Joi.string().required() }

- `generateLetterSchema` — POST /v1/cases/:caseId/report/letter
  - params: { caseId: Joi.string().required() }
  - body: { letterType: Joi.string().valid('qualified_written_request', 'notice_of_error', 'request_for_information').required() }

Export all schemas.

**2. Create Express routes (backend-express/routes/reports.js):**

- `POST /v1/cases/:caseId/report` — Generate consolidated report
  - Validate with generateReportSchema (params + body)
  - Extract userId from req.user.id
  - Call consolidatedReportService.generateReport(caseId, userId, options)
  - If result.error: return 200 with { status: 'error', message: result.errorMessage }
  - Success: return 200 with { status: 'success', report }

- `GET /v1/cases/:caseId/report` — Retrieve latest consolidated report
  - Validate with getReportSchema (params)
  - Query Supabase consolidated_reports table for latest report by caseId + userId
  - Not found: return 404 with { error: 'NotFound', message: 'No consolidated report found' }
  - Success: return 200 with { status: 'success', report }

- `POST /v1/cases/:caseId/report/letter` — Generate dispute letter for existing report
  - Validate with generateLetterSchema (params + body)
  - First retrieve existing consolidated report
  - If no report: return 404
  - Call disputeLetterService.generateDisputeLetter(letterType, report)
  - If letter.error: return 200 with { status: 'error', message }
  - Success: return 200 with { status: 'success', letter }

Follow the exact route handler pattern from compliance.js:
- requireAuth middleware applied to all routes
- validate() middleware for Joi schemas
- try/catch with logger.error and next(error)
- Structured logging at entry and completion

**3. Mount routes in server.js:**
- Add `const reportRoutes = require('./routes/reports');`
- Mount BEFORE case routes: `app.use('/v1/cases', reportRoutes);`
- Position: after compliance routes, before case routes (consistent ordering pattern)
- The report routes use specific paths (/v1/cases/:caseId/report, /v1/cases/:caseId/report/letter) which won't conflict since they're more specific than /:caseId
