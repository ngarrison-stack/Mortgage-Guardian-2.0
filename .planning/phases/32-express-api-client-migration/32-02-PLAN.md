**Express endpoint contracts (from backend routes):**

Claude:
- POST /v1/ai/claude/analyze — body: { documentText, documentType, prompt?, model?, maxTokens?, temperature? } → { success, analysis, model, usage, timestamp }

Documents:
- POST /v1/documents/upload — body: { documentId, fileName, documentType, content (base64), analysisResults?, metadata? } → { success, documentId, storagePath }
- GET /v1/documents — query: { limit?, offset? } → { documents, total, userId }
- GET /v1/documents/:documentId — → document object
- GET /v1/documents/:documentId/analysis — → { documentId, status, analysis }
- DELETE /v1/documents/:documentId — → { success, message }
- POST /v1/documents/process — body: { documentId, documentText, fileBuffer, documentType } → pipeline result
- GET /v1/documents/:documentId/status — → { documentId, step, status, progress }

Plaid:
- POST /v1/plaid/link_token — body: { user_id, client_name?, redirect_uri? } → { link_token, expiration }
- POST /v1/plaid/exchange_token — body: { public_token, user_id, institution_id? } → { access_token, item_id }
- POST /v1/plaid/accounts — body: { access_token, account_ids? } → { accounts, item }
- POST /v1/plaid/transactions — body: { access_token, start_date, end_date, count?, offset? } → { transactions, total_transactions }

**Established patterns:** AWSBackendClient uses Codable structs with CodingKeys for request/response models.
