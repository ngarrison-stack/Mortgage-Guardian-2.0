<task type="auto">
  <name>Task 1: Migrate DocumentManager and DocumentAnalysisService to APIClient</name>
  <files>MortgageGuardian/Services/DocumentManager.swift, MortgageGuardian/Services/DocumentAnalysisService.swift</files>
  <action>
**DocumentManager.swift changes:**

1. Replace manual URL construction (`APIClient.shared.baseURL + "/documents/upload"`) with `APIClient.shared.uploadDocument(documentId:fileName:documentType:content:)` — the new typed method handles URL construction internally
2. Replace `APIClient.shared.request(endpoint: "/documents", ...)` with `APIClient.shared.fetchDocuments()`
3. Replace `APIClient.shared.request(endpoint: "/analysis/ask", ...)` with appropriate typed method or keep generic `request()` if no dedicated method exists for `/analysis/ask`
4. Remove any manual Bearer token header injection — APIClient handles this automatically
5. Update response type references to use the new models from APIClient+Documents.swift

**DocumentAnalysisService.swift changes:**

1. Replace all `AWSBackendClient.shared.analyzeDocumentWithClaude(...)` calls with `APIClient.shared.analyzeDocumentWithClaude(documentText:documentType:)` — note the Express endpoint takes documentText (string), not imageData (base64 image). If DocumentAnalysisService currently sends image data, it needs to send the OCR-extracted text instead (OCR happens locally via Vision Framework on iOS, then text is sent to Express for Claude analysis)
2. Replace `AWSBackendClient.shared.uploadDocumentForTextract(...)` with `APIClient.shared.uploadDocument(...)` — Express doesn't use Textract, it uses its own document pipeline
3. Replace `AWSBackendClient.shared.getDocumentAnalysis(...)` with `APIClient.shared.getDocumentAnalysis(documentId:)`
4. Replace `AWSBackendClient.shared.verifyBankData(...)` — map to appropriate Express Plaid endpoint or keep as-is if Express has a verification endpoint
5. Replace ALL type references: `AWSBackendClient.ClaudeAnalysisResponse` → `ClaudeAnalysisResponse`, `AWSBackendClient.DocumentType` → `DocumentType`, `AWSBackendClient.BankAccountData` → use Plaid models, `AWSBackendClient.AnyCodable` → `AnyCodable`
6. Replace hardcoded AWS health URLs (lines 305, 663) with `APIConfiguration.baseURL + "/health"`

Be careful with the `convertAnyCodableToStringAny` helper — it should work with the migrated AnyCodable type unchanged.
