<task type="auto">
  <name>Task 2: Rewrite APIClient as consolidated networking layer</name>
  <files>MortgageGuardian/Services/APIClient.swift</files>
  <action>
Rewrite `APIClient.swift` to be the single networking client. Consolidate the best parts of both existing clients:

**From current APIClient (keep):**
- Singleton pattern (`APIClient.shared`)
- Generic `request<T: Decodable>()` method with async/await
- `HTTPMethod` enum
- Bearer token auth header injection
- `setAuthToken()` / `authToken` property

**From AWSBackendClient (absorb):**
- Exponential backoff retry logic (max 3 retries, retryable status codes: 429, 500, 502, 503, 504)
- Token refresh on 401 (call back to re-fetch Clerk token)
- Jitter on retry delays

**Changes:**
- Replace `let baseURL = "https://h4rj2gpdza..."` with `var baseURL: String { APIConfiguration.baseURL }`
- Add `private func performRequest<T: Decodable>(...) async throws -> T` with retry logic
- Keep the public `request()` method signature identical so existing callers don't break
- Expand `APIError` enum: add `authenticationRequired`, `serverError(Int, String)`, `networkError(Error)` to match what AWSBackendClient had
- Add `@Observable` macro for SwiftUI compatibility (AWSBackendClient had this)
- Add `var isAuthenticated: Bool { !authToken.isEmpty }` convenience property

**Do NOT add:**
- Cognito-specific code (AWSBackendClient had placeholder Cognito methods — skip all of that)
- Response model types (those stay separate, will be moved in Plan 32-02)

The goal: APIClient becomes a production-quality generic networking layer that any service can use.
  </action>
  <verify>File compiles. `APIClient.shared.baseURL` returns the value from `APIConfiguration`. Retry logic exists. Auth token management works.</verify>
  <done>APIClient.swift is rewritten with configurable URL via APIConfiguration, retry logic with exponential backoff, and expanded error enum. No AWS references remain in APIClient.</done>
</task>
