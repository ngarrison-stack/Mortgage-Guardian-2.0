# 07-02 Plan: Vendor Neutral Security Service Refactoring

## Goal

Break `vendorNeutralSecurityService.js` (827 lines, 13 classes + 1 function) into focused module files while preserving all existing tests.

## Current Structure

13 classes + 1 function in a single file:
1. `VendorNeutralSecurityService` — Main orchestrator (lines 24-197)
2. `NativeEncryptionProvider` — Node.js crypto encryption (lines 199-272)
3. `HashicorpVaultProvider` — Vault transit encryption (lines 277-312)
4. `HardwareHSMProvider` — PKCS#11 placeholder (lines 317-334)
5. `FilesystemSecretManager` — Encrypted file storage (lines 341-388)
6. `DatabaseSecretManager` — DB secret storage (lines 393-411)
7. `KubernetesSecretManager` — K8s secrets (lines 416-431)
8. `DockerSecretManager` — Docker secrets (lines 436-446)
9. `EnvironmentSecretManager` — Env var secrets (lines 451-459)
10. `InMemorySessionManager` — Map-based sessions (lines 466-514)
11. `RedisSessionManager` — Redis sessions (lines 519-568)
12. `ImmutableAuditLog` — Blockchain-style audit (lines 575-683)
13. `ZeroKnowledgeAuth` — SRP authentication (lines 691-766)
14. `createSecurityMiddleware` — Express middleware factory (lines 773-809)

## Target Structure

```
services/
  vendorNeutralSecurity/
    index.js                — Re-export all classes (backward compatible)
    encryptionProviders.js  — NativeEncryptionProvider, HashicorpVaultProvider, HardwareHSMProvider
    secretManagers.js       — FilesystemSecretManager, DatabaseSecretManager, KubernetesSecretManager, DockerSecretManager, EnvironmentSecretManager
    sessionManagers.js      — InMemorySessionManager, RedisSessionManager
    auditLog.js             — ImmutableAuditLog
    zeroKnowledgeAuth.js    — ZeroKnowledgeAuth
    middleware.js           — createSecurityMiddleware
    service.js              — VendorNeutralSecurityService (main orchestrator)
  vendorNeutralSecurityService.js — Keep as thin re-export
```

## Key Constraints

- `require('../../services/vendorNeutralSecurityService')` must return same exports object
- Named exports: { VendorNeutralSecurityService, NativeEncryptionProvider, ... }
- All tests must pass unchanged
- Classes reference each other (e.g., FilesystemSecretManager creates NativeEncryptionProvider)

## Steps

1. Create `services/vendorNeutralSecurity/` directory
2. Extract `encryptionProviders.js` — 3 provider classes
3. Extract `secretManagers.js` — 5 manager classes (imports NativeEncryptionProvider)
4. Extract `sessionManagers.js` — 2 session classes
5. Extract `auditLog.js` — ImmutableAuditLog
6. Extract `zeroKnowledgeAuth.js` — ZeroKnowledgeAuth
7. Extract `middleware.js` — createSecurityMiddleware
8. Extract `service.js` — VendorNeutralSecurityService (imports from other modules for its factory methods)
9. Create `index.js` — re-export everything
10. Update `vendorNeutralSecurityService.js` to be a thin re-export
11. Run full test suite — verify all tests pass
12. Verify no import breakage

## Estimated Duration

~5 min
