# 07-01 Plan: Financial Security Service Refactoring

## Goal

Break `financialSecurityService.js` (848 lines, single class) into focused modules while preserving all 474 existing tests.

## Current Structure

Single `FinancialSecurityService` class with 6 responsibility areas:
1. **Initialization** (lines 82-187): Rate limiters, encryption config, HSM, compliance rules
2. **Credential Management** (lines 189-320): store, retrieve, rotate credentials
3. **Encryption** (lines 322-390): HSM/KMS encrypt/decrypt
4. **Zero-Trust Validation** (lines 392-522): identity, device, network, behavior, permissions
5. **Fraud Detection** (lines 524-579): Transaction risk scoring
6. **Compliance/Audit/Helpers** (lines 581-845): Compliance validation, audit logging, helpers, AWS Secrets Manager, middleware

Plus module-level initialization: AWS SDK clients, Redis, Winston logger (lines 1-76).

## Target Structure

```
services/
  financialSecurity/
    index.js              — Re-export facade (backward compatible)
    credentials.js        — storeCredential, retrieveCredential, rotateCredential
    encryption.js         — encryptWithHSM/KMS, decryptWithHSM/KMS
    validation.js         — validateZeroTrust and sub-validators
    fraud.js              — detectFraud and helpers
    compliance.js         — validateCompliance rules and checking
    audit.js              — auditLog, calculateHash, signAuditEntry
    helpers.js            — sanitizeForLog, generateSecureCredential, verifyMFA, checkSuspiciousActivity
    middleware.js         — securityMiddleware()
    config.js             — AWS clients, Redis, Winston logger, encryption config, compliance rules
  financialSecurityService.js — Keep as thin re-export: module.exports = require('./financialSecurity')
```

## Key Constraints

- `require('../../services/financialSecurityService')` must still work (tests + routes)
- All 474 tests must pass unchanged
- The service exports a singleton instance — the facade must preserve this
- Module-scope AWS/Redis/Winston initialization stays in config.js

## Steps

1. Create `services/financialSecurity/` directory
2. Extract `config.js` — AWS clients, Redis, Winston logger, rate limiters, encryption config, compliance rules
3. Extract `encryption.js` — encrypt/decrypt methods (depend on config for KMS/HSM)
4. Extract `credentials.js` — store/retrieve/rotate (depend on encryption, audit, compliance)
5. Extract `validation.js` — zero-trust validation chain
6. Extract `fraud.js` — detectFraud
7. Extract `compliance.js` — validateCompliance
8. Extract `audit.js` — auditLog, calculateHash, signAuditEntry
9. Extract `helpers.js` — utility methods
10. Extract `middleware.js` — securityMiddleware()
11. Create `index.js` — compose FinancialSecurityService class from modules, export singleton
12. Update `financialSecurityService.js` to be a thin re-export
13. Run full test suite — verify 474 tests pass
14. Verify no import breakage

~8 min
