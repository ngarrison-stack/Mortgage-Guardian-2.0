  <action>
Create tests that inject Claude API failures at each pipeline stage where Claude is called:

**Stage 1 — Classification (classificationService):**
- Claude returns 429 (rate limited) → pipeline should fail with meaningful error, not crash
- Claude returns malformed JSON → pipeline should handle via JSON parse fallback (Phase 10-03 pattern)
- Claude times out (mock rejects with timeout error) → pipeline should fail gracefully

**Stage 2 — Individual Analysis (documentAnalysisService):**
- Claude returns 500 → analysis should return error status, not throw
- Claude returns empty response → analysis should handle gracefully
- Claude returns valid JSON but missing required fields → schema validation as warnings (Phase 12-02)

**Stage 3 — Cross-Document Comparison (crossDocumentComparisonService):**
- Claude fails mid-comparison (some pairs succeed, some fail) → forensic orchestrator should include successful pairs and log failures
- Claude returns inconsistent severity values → result enrichment should validate/fix enums (Phase 13-03)

**Stage 4 — Compliance AI Enhancement (complianceAnalysisService):**
- Claude fails entirely → original rule-engine violations returned unchanged (Phase 14-04)
- Claude returns partial enhancement → graceful merge of enhanced + unenhanced

For each test: mock the Anthropic SDK to reject/return bad data, call the orchestrator, verify the response is an error object (not a thrown exception), and verify the error message is meaningful (not "undefined" or empty).

Use the mockPipelineServices factory from Plan 01 as the baseline, then override specific Claude mock behaviors per test.
