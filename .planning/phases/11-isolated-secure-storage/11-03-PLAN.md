# Existing service patterns:
@backend-express/services/documentService.js

**Tech stack available:** Node.js 20+ (crypto module built-in), Jest 29.x
**Established patterns:** Singleton service pattern (`class X { } module.exports = new X()`), createLogger child loggers

**Design decisions:**
- **Algorithm:** AES-256-GCM (authenticated encryption — tamper detection built in)
- **Key derivation:** HKDF-SHA256 from master key + userId salt → 32-byte per-user key
- **Master key:** From DOCUMENT_ENCRYPTION_KEY env var (hex-encoded 32-byte key)
- **IV:** Random 12 bytes per encrypt operation (crypto.randomBytes(12))
- **Pack format:** Single buffer = iv(12) + authTag(16) + ciphertext(N) — unpack by splitting at fixed offsets
- **No external dependencies:** Node.js crypto module only (built-in since Node 15+)
</context>

<feature>
  <name>Document Encryption Service</name>
  <files>backend-express/services/documentEncryptionService.js, backend-express/__tests__/services/documentEncryptionService.test.js</files>
  <behavior>
    **Methods:**
    - `deriveKey(userId)` → 32-byte Buffer (deterministic for same userId + master key)
    - `encrypt(userId, plaintextBuffer)` → Buffer (packed: iv + authTag + ciphertext)
    - `decrypt(userId, encryptedBuffer)` → Buffer (original plaintext)

    **Test cases (input → expected output):**
    1. encrypt then decrypt returns original data (round-trip)
    2. Different users get different derived keys
    3. Same user always gets same derived key (deterministic)
    4. Tampered ciphertext throws authentication error
    5. Tampered authTag throws authentication error
    6. Wrong userId for decrypt throws error (wrong key)
    7. Missing DOCUMENT_ENCRYPTION_KEY env var throws descriptive error
    8. Empty buffer encrypt/decrypt works (0-byte files)
    9. Large buffer (1MB) encrypt/decrypt works correctly
    10. Encrypted output is different from plaintext (not passthrough)
    11. Pack format: output length = 12 + 16 + plaintextLength (for GCM with no padding)
  </behavior>
  <implementation>
    Using Node.js crypto module (zero dependencies):

    ```
    deriveKey(userId):
      masterKeyBuffer = Buffer.from(process.env.DOCUMENT_ENCRYPTION_KEY, 'hex')
      return crypto.hkdfSync('sha256', masterKeyBuffer, userId, 'mortgage-guardian-doc-v1', 32)

    encrypt(userId, plaintext):
      key = deriveKey(userId)
      iv = crypto.randomBytes(12)
      cipher = crypto.createCipheriv('aes-256-gcm', key, iv)
      ciphertext = Buffer.concat([cipher.update(plaintext), cipher.final()])
      authTag = cipher.getAuthTag()  // 16 bytes
      return Buffer.concat([iv, authTag, ciphertext])

    decrypt(userId, packed):
      key = deriveKey(userId)
      iv = packed.subarray(0, 12)
      authTag = packed.subarray(12, 28)
      ciphertext = packed.subarray(28)
      decipher = crypto.createDecipheriv('aes-256-gcm', key, iv)
      decipher.setAuthTag(authTag)
      return Buffer.concat([decipher.update(ciphertext), decipher.final()])
    ```

    Service follows singleton pattern matching documentService.js.
    Use createLogger('encryption') for structured logging.
    Log encrypt/decrypt operations at debug level (NEVER log keys or plaintext).
  </implementation>
</feature>
