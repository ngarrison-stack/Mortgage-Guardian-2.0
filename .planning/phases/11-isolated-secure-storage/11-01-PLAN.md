<objective>
Add Row Level Security (RLS) policies to all document-related database tables to enforce per-user data isolation at the database level.

Purpose: Defense-in-depth — even if application-level auth is bypassed or the service key is compromised, the database enforces that users can only access their own data. Critical for a platform handling sensitive mortgage documents. Migration 002 explicitly deferred RLS to this phase.
Output: Migration 004 with RLS policies on documents, case_files, document_classifications, pipeline_state; RLS verification tests confirming policy enforcement.
</objective>

**Architecture note:** Backend uses SUPABASE_SERVICE_KEY which bypasses RLS. These policies serve as:
1. Defense-in-depth if service key is compromised
2. Protection for any direct Supabase client access (iOS/frontend)
3. Documentation of access control intent
4. Foundation for future per-request user-scoped Supabase clients

  <name>Task 1: Create RLS migration for all document tables</name>
  <files>backend-express/migrations/004_document_rls_policies.sql</files>
  <action>
Create migration 004 that enables RLS and creates user-scoped policies for four tables:

**1. documents table:**
- Enable RLS: `ALTER TABLE documents ENABLE ROW LEVEL SECURITY;`
- SELECT policy: users can view own documents (`auth.uid()::text = user_id`)
- INSERT policy: users can insert own documents (`auth.uid()::text = user_id`)
- UPDATE policy: users can update own documents (`auth.uid()::text = user_id`)
- DELETE policy: users can delete own documents (`auth.uid()::text = user_id`)

**2. case_files table:**
- Enable RLS (already created in 002 without RLS)
- Same 4 CRUD policies scoped to `user_id`

**3. document_classifications table:**
- Enable RLS
- SELECT policy: join through documents table to verify ownership (`EXISTS (SELECT 1 FROM documents d WHERE d.document_id = document_classifications.document_id AND d.user_id = auth.uid()::text)`)
- INSERT policy: same join check
- Note: document_classifications does NOT have its own user_id column — it links through document_id. Policy must join to documents table.

**4. pipeline_state table:**
- Enable RLS
- Same 4 CRUD policies scoped to `user_id`

**Policy naming convention:** Follow migration 001 pattern — descriptive names like "Users can view own documents".

**Use IF NOT EXISTS/DO $$ guard** for each policy to make migration idempotent (safe to re-run). Wrap each CREATE POLICY in a DO block that checks pg_policies first, same pattern as the trigger guard in migration 002.

**Do NOT** create a service_role bypass policy — service_role key already bypasses RLS by default in Supabase.
  </action>
