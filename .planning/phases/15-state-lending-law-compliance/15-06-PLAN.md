<feature>
  <name>State Compliance Rule Engine Evaluation</name>
  <files>backend-express/services/complianceRuleEngine.js, backend-express/__tests__/complianceRuleEngine.test.js</files>
  <behavior>
    evaluateStateFindings(forensicReport, analysisReports, jurisdiction) → { stateViolations[], stateStatutesEvaluated[], evaluationMeta }

    Cases:
    - jurisdiction.applicableStates = ['CA'], discrepancy with escrow keywords → CA HBOR/escrow violation produced
    - jurisdiction.applicableStates = ['NY', 'TX'], findings match NY and TX rules → violations from both states
    - jurisdiction.applicableStates = ['CA'], but no findings match CA rules → empty stateViolations, CA statutes still in stateStatutesEvaluated
    - jurisdiction.applicableStates = [] → empty result, no states evaluated
    - null/missing jurisdiction → error object returned
    - Unsupported state in applicableStates → skipped gracefully, logged
    - Violation IDs: 'state-viol-001', 'state-viol-002' (separate from federal 'viol-001')
    - Deduplication: same state section + same evidence source → keep higher severity
    - State violations include `jurisdiction` field (state code) in each violation object
    - Multiple states can produce violations for same finding (e.g., both CA and federal escrow)
  </behavior>
  <implementation>Add evaluateStateFindings() method to existing ComplianceRuleEngine class. Reuse existing _evaluateDiscrepancies, _evaluateAnomalies, etc. normalization methods. Use matchStateRules() from stateComplianceRuleMappings instead of federal matchRules(). Use getStateStatuteById/getStateSectionById from stateStatuteTaxonomy for violation building. Add jurisdiction field to state violation objects. Keep state violation ID sequence separate from federal (prefix 'state-viol-').</implementation>
</feature>
