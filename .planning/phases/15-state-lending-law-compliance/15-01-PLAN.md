<task type="auto">
  <name>Task 2: Create state statute taxonomy configuration with helpers</name>
  <files>backend-express/config/stateStatuteTaxonomy.js</files>
  <action>Create `stateStatuteTaxonomy.js` mirroring the structure of `federalStatuteTaxonomy.js` but organized by state:

1. Define `STATE_STATUTES` object keyed by 2-letter state code (e.g., `CA`, `NY`, `TX`, `FL`, `IL`, `MA`)
   - Each state entry has: `{ stateCode, stateName, statutes: {} }`
   - Each statute within a state: `{ id, name, citation, enforcementBody, sections: [] }`
   - Each section: `{ id, section, title, regulatoryReference, requirements: [], violationPatterns: [], penalties }`
   - violationPatterns follow the same `{ discrepancyType, anomalyType, keywords, severity }` shape as federal

2. Start with empty state entries for priority states: CA, NY, TX, FL, IL, MA (statutes array empty — populated in 15-03/15-04)

3. Create helper functions mirroring federal taxonomy helpers:
   - `getStateStatutes(stateCode)` — returns all statutes for a state
   - `getStateStatuteById(stateCode, statuteId)` — returns specific statute
   - `getStateSectionById(stateCode, sectionId)` — searches all state statutes for a section
   - `getSupportedStates()` — returns array of state codes with statute data
   - `getStateStatuteIds(stateCode)` — returns array of statute IDs for a state
   - `isStateSupported(stateCode)` — boolean check

4. Export: STATE_STATUTES, all helper functions

Use the SAME data shape as federalStatuteTaxonomy.js so the existing matchRules() function from complianceRuleMappings.js can process state rules identically to federal ones.</action>
  <verify>Run `node -e "const s = require('./backend-express/config/stateStatuteTaxonomy'); console.log(s.getSupportedStates()); console.log(s.isStateSupported('CA'));"` — should list 6 states and return true for CA.</verify>
  <done>stateStatuteTaxonomy.js created with STATE_STATUTES structure, 6 empty priority state entries, and 6 helper functions. Data shape matches federalStatuteTaxonomy.js.</done>
