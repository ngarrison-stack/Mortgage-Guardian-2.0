<task type="auto">
  <name>Task 1: Create state compliance rule mappings for priority states</name>
  <files>backend-express/config/stateComplianceRuleMappings.js</files>
  <action>Create stateComplianceRuleMappings.js following the exact structure of complianceRuleMappings.js:

1. Define STATE_COMPLIANCE_RULE_MAPPINGS as an object keyed by state code:
   ```
   { CA: [...rules], NY: [...rules], TX: [...rules], FL: [...rules], IL: [...rules], MA: [...rules] }
   ```

2. Each rule follows the identical shape as federal rules:
   - ruleId: 'rule-{stateCode}-{statute}-{nnn}' (e.g., 'rule-ca-hbor-001')
   - sectionId: matches section IDs from stateStatuteTaxonomy.js
   - category: same categories as federal (escrow, disclosure, collection, etc.)
   - matchCriteria: { discrepancyTypes, anomalyTypes, timelineViolation, paymentIssue, keywords, fieldPatterns, minSeverity }
   - violationSeverity, severityElevation, descriptionTemplate, legalBasisTemplate

3. Target rule counts per state:
   - CA: ~8 rules (HBOR dual tracking, escrow, disclosure, collection)
   - NY: ~6 rules (RPAPL foreclosure, servicing, consumer protection)
   - TX: ~5 rules (foreclosure notice, lending, collection)
   - FL: ~5 rules (foreclosure, collection, lending)
   - IL: ~5 rules (foreclosure notice, lending, consumer fraud)
   - MA: ~5 rules (predatory lending, right to cure, consumer protection)

4. Create helper functions mirroring complianceRuleMappings.js:
   - `getStateRules(stateCode)` — returns rules array for a state
   - `getStateRulesForSection(stateCode, sectionId)` — filter by section
   - `matchStateRules(stateCode, finding)` — same matching logic as federal matchRules() but state-scoped
   - `getSupportedRuleStates()` — states with rules defined

5. Export: STATE_COMPLIANCE_RULE_MAPPINGS, helper functions

IMPORTANT: The matchStateRules() function should use IDENTICAL matching logic to the federal matchRules() — same keyword matching, same severity comparison, same fieldPattern matching. Consider extracting the common matching logic into a shared helper if it reduces duplication significantly, but only if it's clean — don't over-abstract.</action>
  <verify>Run `node -e "const s = require('./backend-express/config/stateComplianceRuleMappings'); console.log('States:', s.getSupportedRuleStates()); console.log('CA rules:', s.getStateRules('CA').length);"` — should show 6 states and ~8 CA rules.</verify>
  <done>stateComplianceRuleMappings.js with ~34 rules across 6 states, matchStateRules() function, helper exports. Rule shape identical to federal rules.</done>
</task>
