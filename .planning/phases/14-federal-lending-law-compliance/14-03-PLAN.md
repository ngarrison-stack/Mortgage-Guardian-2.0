<feature>
  <name>Compliance Rule Engine — evaluateFindings()</name>
  <files>backend-express/services/complianceRuleEngine.js, backend-express/__tests__/complianceRuleEngine.test.js</files>
  <behavior>
    The compliance rule engine evaluates forensic analysis data against federal statute rules.

    **Primary method: evaluateFindings(forensicReport, analysisReports)**
    - Input: forensicReport (cross-document analysis output), analysisReports (array of individual document analyses)
    - Output: { violations: Violation[], statutesEvaluated: string[], evaluationMeta: {} }

    **Cases:**
    1. Discrepancy with matching rule → produces violation with correct statute/section/citation
    2. Anomaly with matching rule → produces violation linked to anomaly evidence
    3. Timeline violation with matching rule → produces timeline-based violation
    4. Payment verification issue with matching rule → produces payment-based violation
    5. Finding below minSeverity threshold → no violation produced
    6. Multiple rules match same finding → produces multiple violations (one per rule)
    7. Severity elevation conditions met → violation severity elevated
    8. No findings match any rules → empty violations array, all statutes still listed as evaluated
    9. Empty forensicReport → returns empty violations with all statutes evaluated
    10. Missing/null inputs → returns error object (consistent with forensicAnalysisService pattern)

    **Helper methods to test:**
    - _evaluateDiscrepancies(discrepancies) → violations[]
    - _evaluateAnomalies(anomalies) → violations[]
    - _evaluateTimelineViolations(violations) → violations[]
    - _evaluatePaymentIssues(paymentVerification) → violations[]
    - _buildViolation(finding, rule, evidence) → violation object
    - _shouldElevateSeverity(finding, rule) → boolean
  </behavior>
  <implementation>
    Class ComplianceRuleEngine with singleton export pattern (consistent with other services).

    evaluateFindings():
    1. Validate inputs (caseId, forensicReport exist)
    2. Extract discrepancies, anomalies, timeline violations, payment issues from forensicReport
    3. Extract anomalies from individual analysisReports
    4. Run each through respective _evaluate* methods
    5. Assign sequential violation IDs (viol-001, viol-002, etc.)
    6. Deduplicate violations by sectionId + evidence sourceId (keep higher severity)
    7. Return { violations, statutesEvaluated: all statute ids, evaluationMeta }

    _buildViolation(finding, rule, evidence):
    1. Look up statute and section from federalStatuteTaxonomy
    2. Fill descriptionTemplate with finding data
    3. Fill legalBasisTemplate with finding data
    4. Apply severity elevation if conditions met
    5. Return violation object matching complianceReportSchema

    Use matchRules() from complianceRuleMappings to find applicable rules per finding.
    Use getStatuteById()/getSectionById() from federalStatuteTaxonomy for citation data.
  </implementation>
</feature>
