<task type="auto">
  <name>Task 1: Create compliance analysis Claude AI service</name>
  <files>backend-express/services/complianceAnalysisService.js</files>
  <action>
Create a singleton service that uses Claude AI to generate legal analysis for compliance violations.

**Class: ComplianceAnalysisService**

**Methods:**

1. `async analyzeViolations(violations, caseContext)` — Main method
   - Input: violations array (from rule engine), caseContext { caseId, documentTypes, discrepancySummary }
   - Output: { enhancedViolations: [], legalNarrative: string, analysisMetadata: {} }
   - For each violation batch (group by statuteId), call Claude with statute-specific prompt
   - Claude enhances each violation with: detailed legalBasis, potentialPenalties, recommendations
   - Generate overall legalNarrative summarizing all violations
   - Use claude-sonnet-4-5 with temperature 0.1

2. `async generateLegalNarrative(violations, caseContext)` — Generate case-level summary
   - Input: enhanced violations array, case context
   - Output: string (markdown-formatted legal narrative)
   - Prompt Claude to produce a 3-5 paragraph legal summary suitable for a compliance report
   - Include: overview of violations found, most serious concerns, regulatory implications, recommended actions

3. `_buildCompliancePrompt(violations, statute, caseContext)` — Build statute-specific prompt
   - Load statute details from federalStatuteTaxonomy
   - Include relevant sections, requirements, and penalties
   - Structure prompt to get JSON output with enhanced violation details
   - Include instruction to cite specific regulatory provisions

4. `_buildNarrativePrompt(enhancedViolations, caseContext)` — Build narrative prompt
   - Structure prompt for markdown narrative generation
   - Include all violation summaries as context
   - Instruct for professional legal tone

5. `_parseClaudeResponse(response)` — Parse Claude JSON response
   - Extract JSON from response text
   - Handle markdown code fence wrapping (```json ... ```)
   - Return parsed object or { rawResponse, parseError } on failure (consistent with 12-02 pattern)

**Implementation details:**
- Lazy Anthropic client initialization (consistent with 10-02 pattern)
- Temperature 0.1 for deterministic legal analysis
- Model: claude-sonnet-4-5 (consistent with 10-03, 13-03)
- Max tokens: 4096 for violations, 2048 for narrative
- Graceful error handling: if Claude call fails, return violations unchanged with warning
- Log analysis duration and token usage
- Batch violations by statute to avoid context overflow (max 10 violations per Claude call)

Use Winston logger with `createLogger('compliance-analysis')`.
