  <name>Task 1: Create compliance orchestrator service</name>
  <files>backend-express/services/complianceService.js</files>
  <action>
Create a singleton ComplianceService that orchestrates the full federal lending law compliance analysis.

**Class: ComplianceService**

**Main method: `async evaluateCompliance(caseId, userId, options = {})`**

Follow the exact pattern from forensicAnalysisService.analyzeCaseForensics():

**Step 1: GATHER FORENSIC DATA**
- Retrieve the forensic analysis report for this case from caseFileService
- Retrieve individual document analysis reports from the pipeline
- If no forensic report exists, return error: "Case has no forensic analysis. Run forensic analysis first."
- Track step timing and status in stepMeta

**Step 2: RULE ENGINE EVALUATION**
- Call complianceRuleEngine.evaluateFindings(forensicReport, analysisReports)
- Capture violations and evaluation metadata
- If rule engine throws, catch error and return partial result with warning
- Track step timing

**Step 3: CLAUDE AI ENHANCEMENT (optional, controlled by options.skipAiAnalysis)**
- If options.skipAiAnalysis is true, skip this step (allows fast rule-only evaluation)
- Call complianceAnalysisService.analyzeViolations(violations, caseContext)
- Merge enhanced violations back (replace legalBasis, add recommendations, potentialPenalties)
- Call complianceAnalysisService.generateLegalNarrative(enhancedViolations, caseContext)
- If Claude analysis fails, keep original rule-engine violations with warning
- Track step timing

**Step 4: ASSEMBLE REPORT**
- Build compliance report matching complianceReportSchema:
  - caseId, analyzedAt, statutesEvaluated, violations, complianceSummary, legalNarrative
- Calculate complianceSummary:
  - totalViolations, criticalViolations, highViolations
  - statutesViolated (unique statute IDs from violations)
  - overallComplianceRisk (critical if any critical, high if any high, etc.)
  - keyFindings (top 10 violations by severity)
  - recommendations (deduplicated from all violations)
- Validate report against complianceReportSchema (warnings only, don't reject)
- Attach _metadata with duration, steps, warnings

**Step 5: PERSIST (best-effort)**
- Save compliance report to caseFileService.updateCase({ compliance_report: report })
- Log warning on failure, never throw (consistent with forensicAnalysisService._persistReport)

**Input validation:**
- caseId required, userId required (return error object if missing, don't throw)

**Options:**
- skipAiAnalysis: boolean (default false) — skip Claude enhancement for fast evaluation
- statuteFilter: string[] — only evaluate specific statutes (default: all)

**Error handling:**
- Same pattern as forensicAnalysisService: return error objects, don't throw
- Accumulate warnings array throughout
- Each step has try/catch with graceful degradation

Use Winston logger: `createLogger('compliance')`.
