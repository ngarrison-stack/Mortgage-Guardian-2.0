<task type="auto">
  <name>Task 2: Create compliance API routes and register in server</name>
  <files>backend-express/routes/compliance.js, backend-express/server.js</files>
  <action>
Create compliance API routes following the established patterns from documents.js and cases.js.

**Routes in compliance.js:**

1. `POST /v1/cases/:caseId/compliance` — Run compliance evaluation
   - Auth: requireAuth (userId from req.user.id)
   - Validation: evaluateComplianceSchema
   - Handler: Call complianceService.evaluateCompliance(caseId, userId, options)
   - Response: 200 with compliance report, or 200 with { status: 'error', message } on failure
   - Error: 500 for unexpected errors

2. `GET /v1/cases/:caseId/compliance` — Get existing compliance report
   - Auth: requireAuth
   - Handler: Retrieve stored compliance report from caseFileService
   - Response: 200 with report, 404 if no report exists

3. `GET /v1/compliance/statutes` — List all federal statutes
   - Auth: requireAuth
   - Handler: Return federalStatuteTaxonomy statute list
   - Response: 200 with { statutes: [...] }

4. `GET /v1/compliance/statutes/:statuteId` — Get statute details
   - Auth: requireAuth
   - Handler: Return specific statute from taxonomy
   - Response: 200 with statute details, 404 if not found

**Server.js registration:**
- Add compliance routes: `app.use('/v1', requireAuth, complianceRoutes);`
- Register compliance-specific routes BEFORE case parameterized routes (/:caseId) if they share the /v1/cases path
- Actually, since compliance routes under /v1/cases/:caseId/compliance and /v1/compliance/, register as separate router

**IMPORTANT:** The /v1/compliance/statutes routes must be mounted so they don't conflict with /v1/cases/:caseId. Follow the 12-03/13-06 pattern of specific routes before parameterized routes.

Use Winston logger: `createLogger('compliance-routes')`.
  </action>
  <verify>node -e "require('./backend-express/routes/compliance'); console.log('Routes loaded');" && grep -n "compliance" backend-express/server.js</verify>
  <done>Routes load without error. Server.js registers compliance routes. No route conflicts with existing case routes. All 4 endpoints defined with proper auth and validation.</done>
</task>
