  <name>Task 1: Create document endpoint Joi schemas</name>
  <files>backend-express/schemas/documents.js</files>
  <action>
    Create schemas for all 4 document endpoints:

    1. `uploadDocumentSchema` (body) for POST /v1/documents/upload:
       - `documentId`: string, required, trimmed
       - `userId`: string, required, trimmed
       - `fileName`: string, required, trimmed
       - `content`: string, required (document content — can be large, no max length)
       - `documentType`: string, optional, trimmed, default 'unknown'
       - `analysisResults`: object, optional (passthrough — no deep validation needed)
       - `metadata`: object, optional (passthrough)

    2. `getDocumentsSchema` (query) for GET /v1/documents:
       - `userId`: string, required, trimmed
       - `limit`: number, optional, integer, min(1), max(500), default(50)
       - `offset`: number, optional, integer, min(0), default(0)

    3. `getDocumentSchema` (query) for GET /v1/documents/:documentId:
       - `userId`: string, required, trimmed

    4. `deleteDocumentSchema` (query) for DELETE /v1/documents/:documentId:
       - `userId`: string, required, trimmed

    Use CommonJS exports. Export each schema as a named export.
    Do NOT use Joi.alternatives() or complex conditional logic — keep schemas simple and flat.
  </action>
  <verify>`node -e "const s = require('./backend-express/schemas/documents.js'); console.log(Object.keys(s));"` prints array with 4 schema names</verify>
  <done>documents.js exports 4 Joi schemas covering all document endpoint inputs.</done>

<task type="auto">
  <name>Task 2: Wire schemas into document routes</name>
  <files>backend-express/routes/documents.js</files>
  <action>
    Replace inline validation in documents.js with Joi middleware:

    1. Add imports at top:
       - `const { validate } = require('../middleware/validate');`
       - `const { uploadDocumentSchema, getDocumentsSchema, getDocumentSchema, deleteDocumentSchema } = require('../schemas/documents');`

    2. Wire schemas as middleware on each route:
       - `router.post('/upload', validate(uploadDocumentSchema), async (req, res, next) => { ... })`
       - `router.get('/', validate(getDocumentsSchema, 'query'), async (req, res, next) => { ... })`
       - `router.get('/:documentId', validate(getDocumentSchema, 'query'), async (req, res, next) => { ... })`
       - `router.delete('/:documentId', validate(deleteDocumentSchema, 'query'), async (req, res, next) => { ... })`

    3. REMOVE the inline validation blocks from each handler:
       - POST /upload: Remove the `if (!documentId || !userId || !fileName || !content)` block (lines 19-25)
       - GET /: Remove the `if (!userId)` block (lines 58-63)
       - GET /:documentId: Remove the `if (!userId)` block (lines 90-95)
       - DELETE /:documentId: Remove the `if (!userId)` block (lines 127-132)

    4. After Joi validation, req.body and req.query are already validated — handlers can destructure directly without null checks.

    5. Keep the rest of each handler unchanged (service calls, error handling, response format).
  </action>
