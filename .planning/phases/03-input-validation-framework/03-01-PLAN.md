<task type="auto">
  <name>Task 1: Create Joi validation middleware factory</name>
  <files>backend-express/middleware/validate.js</files>
  <action>
    Create a validation middleware factory function that:

    1. Exports `validate(schema, source)` where:
       - `schema` is a Joi schema object
       - `source` is optional, defaults to 'body' — can be 'body', 'query', or 'params'
    2. Returns Express middleware `(req, res, next)` that:
       - Calls `schema.validate(req[source], { abortEarly: false, stripUnknown: true })`
       - `abortEarly: false` collects ALL validation errors (not just first)
       - `stripUnknown: true` removes fields not in schema (security: prevents injection of unexpected fields)
       - On validation error: returns 400 with `{ error: 'Bad Request', message: formatted_errors }` matching existing error format
       - Format errors as comma-separated list of Joi detail messages (e.g., "\"user_id\" is required, \"count\" must be a number")
       - On success: replaces `req[source]` with the validated/coerced value from `schema.validate()` (Joi coerces types like string→number), then calls `next()`
    3. Use CommonJS `module.exports` (project convention)
    4. Do NOT add logging — validation is a hot path, keep it lean
    5. Keep it simple — no custom error formatting beyond what Joi provides, no error codes, no field-level error arrays
  </action>
  <verify>
    `node -e "const v = require('./backend-express/middleware/validate.js'); const Joi = require('joi'); const schema = Joi.object({ name: Joi.string().required() }); console.log(typeof v.validate === 'function'); const mw = v.validate(schema); console.log(typeof mw === 'function'); console.log('OK');"` prints true, true, OK
  </verify>
  <done>validate.js exports a `validate` function that returns Express middleware. Middleware validates req[source] against Joi schema, returns 400 on failure with `{ error, message }` format, replaces req[source] with validated value on success.</done>
</task>
