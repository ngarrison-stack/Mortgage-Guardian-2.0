<task type="auto">
  <name>Task 1: Create Plaid endpoint Joi schemas</name>
  <files>backend-express/schemas/plaid.js</files>
  <action>
    Create schemas for all validated Plaid endpoints (excluding webhook — it uses raw body + signature verification, and test — no body needed):

    1. `linkTokenSchema` (body) for POST /v1/plaid/link_token:
       - `user_id`: string, required, trimmed, max(255)
       - `client_name`: string, optional, trimmed
       - `redirect_uri`: string, optional, trimmed, uri()
       - `access_token`: string, optional, trimmed
       - `products`: array of strings, optional

    2. `exchangeTokenSchema` (body) for POST /v1/plaid/exchange_token:
       - `public_token`: string, required, trimmed, pattern(/^public-/) with message 'Invalid public token format'
       - `user_id`: string, optional, trimmed
       - `institution_id`: string, optional, trimmed

    3. `accountsSchema` (body) for POST /v1/plaid/accounts:
       - `access_token`: string, required, trimmed, pattern(/^access[-_]/) with message 'Invalid access token format'
       - `account_ids`: array of strings, optional

    4. `transactionsSchema` (body) for POST /v1/plaid/transactions:
       - `access_token`: string, required, trimmed, pattern(/^access[-_]/) with message 'Invalid access token format'
       - `start_date`: string, required, pattern(/^\d{4}-\d{2}-\d{2}$/) with message 'Dates must be in YYYY-MM-DD format'
       - `end_date`: string, required, pattern(/^\d{4}-\d{2}-\d{2}$/) with message 'Dates must be in YYYY-MM-DD format'
       - `account_ids`: array of strings, optional
       - `count`: number, optional, integer, min(1), max(500), default(100)
       - `offset`: number, optional, integer, min(0), default(0)

    5. `itemSchema` (body) for POST /v1/plaid/item:
       - `access_token`: string, required, trimmed, pattern(/^access[-_]/) with message 'Invalid access token format'

    6. `updateWebhookSchema` (body) for POST /v1/plaid/item/webhook:
       - `access_token`: string, required, trimmed, pattern(/^access[-_]/) with message 'Invalid access token format'
       - `webhook`: string, required, trimmed, uri()

    7. `deleteItemSchema` (body) for DELETE /v1/plaid/item:
       - `access_token`: string, required, trimmed, pattern(/^access[-_]/) with message 'Invalid access token format'

    8. `sandboxTokenSchema` (body) for POST /v1/plaid/sandbox_public_token:
       - `institution_id`: string, optional, trimmed
       - `initial_products`: array of strings, optional

    Note on access_token pattern: Use `/^access[-_]/` to match both `access-` (standard) and `access_sandbox-` formats. This replaces the inline `startsWith('access-') && !startsWith('access_sandbox-')` check.

    Use CommonJS exports. Export each schema as a named export.
  </action>
  <verify>`node -e "const s = require('./backend-express/schemas/plaid.js'); console.log(Object.keys(s).length + ' schemas');"` prints "8 schemas"</verify>
  <done>plaid.js exports 8 Joi schemas covering all validated Plaid endpoint inputs.</done>
</task>

<task type="auto">
  <name>Task 2: Wire schemas into Plaid routes</name>
  <files>backend-express/routes/plaid.js</files>
  <action>
    Replace validateFields and inline validation in plaid.js with Joi middleware:

    1. Add imports at top (after existing requires):
       - `const { validate } = require('../middleware/validate');`
       - Import all 8 schemas from `'../schemas/plaid'`

    2. REMOVE the `validateFields` function (lines 13-30) — completely replaced by Joi validate middleware.
       Keep the `sanitizeInput` middleware (lines 35-51) and `router.use(sanitizeInput)` (line 54) — sanitization is a separate concern from validation.

    3. Wire schemas as middleware on each route, replacing `validateFields([...])`:
       - `router.post('/link_token', validate(linkTokenSchema), async ...)`
       - `router.post('/exchange_token', validate(exchangeTokenSchema), async ...)`
       - `router.post('/accounts', validate(accountsSchema), async ...)`
       - `router.post('/transactions', validate(transactionsSchema), async ...)`
       - `router.post('/item', validate(itemSchema), async ...)`
       - `router.post('/item/webhook', validate(updateWebhookSchema), async ...)`
       - `router.delete('/item', validate(deleteItemSchema), async ...)`
       - `router.post('/sandbox_public_token', validate(sandboxTokenSchema), async ...)`

    4. REMOVE inline validation from each handler (Joi now handles all of it):
       - link_token: Remove `user_id.length > 255` check and `products` array check
       - exchange_token: Remove `public_token.startsWith('public-')` check
       - accounts: Remove `access_token.startsWith(...)` check and `account_ids` array check
       - transactions: Remove `access_token.startsWith(...)` check, date regex check, count range check, offset check, account_ids array check
       - item: Remove `access_token.startsWith(...)` check
       - item/webhook: Remove `access_token.startsWith(...)` check and `new URL(webhook)` check
       - delete/item: Remove `access_token.startsWith(...)` check

    5. Leave webhook endpoint (/webhook) and test endpoint (/test) unchanged — webhook uses raw body, test has no inputs.

    6. Keep service calls, error handling, and response formatting unchanged.

<task type="auto">
  <name>Task 3: Verify Plaid webhook URL validation edge case</name>
  <files>None (verification only)</files>
  <action>
    The original `/item/webhook` handler had production-only HTTPS enforcement (`url.protocol !== 'https:' && process.env.NODE_ENV === 'production'`). The Joi uri() validator accepts both http and https. This is acceptable because:
    - Development/sandbox environments need HTTP for local testing
    - HTTPS enforcement in production should be at the infrastructure level (load balancer, Plaid's own requirements)
    - The original check only applied in production, not in development

    Verify this is not a regression by confirming the Joi schema accepts valid URIs:
    `node -e "const Joi = require('joi'); const s = Joi.string().uri(); console.log(s.validate('https://example.com/webhook').error); console.log(s.validate('http://localhost:3000/webhook').error); console.log(s.validate('not-a-url').error ? 'invalid URL rejected' : 'BUG');"` should print null, null, "invalid URL rejected"
  </action>
  <verify>Joi uri() accepts https and http URLs, rejects non-URLs</verify>
  <done>Webhook URL validation behaves correctly for both development (http) and production (https) environments.</done>
