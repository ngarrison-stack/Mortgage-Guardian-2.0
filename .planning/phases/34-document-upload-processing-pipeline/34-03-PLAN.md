<task type="auto">
  <name>Task 1: Add periodic status polling for documents in pipeline</name>
  <files>MortgageGuardian/Views/DocumentsView.swift, MortgageGuardian/Models/UserStore.swift</files>
  <action>
1. In UserStore (or a new lightweight service if UserStore is too large), add a method `pollDocumentStatuses()` that:
   - Filters documents where isAnalyzed == false AND serverDocumentId != nil (documents in the pipeline)
   - For each, calls APIClient.shared.getDocumentStatus(documentId:)
   - Updates the document's status/isAnalyzed based on response
   - When status reaches "analyzed" or "complete", marks isAnalyzed = true and stops polling that document

2. Add a Timer-based polling mechanism (similar to AuthManager's periodic token refresh):
   - Start polling when DocumentsView appears and there are in-pipeline documents
   - Poll every 5 seconds (backend pipeline steps take seconds each)
   - Stop polling when no documents are in-pipeline or view disappears
   - Use Task with cancellation support so polling stops cleanly on view dismiss

3. Store the current pipeline status as a string on MortgageDocument (add a `pipelineStatus: String?` property if needed), so the UI can display it.

Do NOT poll for documents that are already "complete" or "analyzed". Do NOT poll if there are no in-pipeline documents. Cancel polling tasks on view disappear to avoid background resource usage.
  </action>
  <verify>Build succeeds. Polling starts when view has in-pipeline documents. Polling stops when all documents complete or view disappears.</verify>
  <done>Status polling works with 5-second interval. Documents transition from "uploaded" through pipeline states to "analyzed/complete". Polling auto-stops when no in-pipeline documents remain.</done>
</task>

<task type="auto">
  <name>Task 2: Show processing progress indicator in DocumentsView</name>
  <files>MortgageGuardian/Views/DocumentsView.swift</files>
  <action>
1. In the document row view within DocumentsView, check the document's pipelineStatus:
   - If status is nil or "complete" or "analyzed": show existing static UI (analyzed badge, etc.)
   - If status is "uploaded", "ocr", "classifying", "analyzing": show a ProgressView() spinner with a human-readable status label

2. Map pipeline status strings to user-friendly labels:
   - "uploaded" → "Queued for processing..."
   - "ocr" → "Extracting text..."
   - "classifying" → "Identifying document type..."
   - "analyzing" → "Analyzing for issues..."
   - "analyzed" → "Analysis complete"
   - "review" → "Ready for review"
   - "complete" → show green checkmark badge

3. Use a subtle animation (opacity or scale) when status changes to draw attention to progress updates.

Keep the existing document row layout — add the progress indicator BELOW the existing filename/type/date info, not replacing it. Use .font(.caption) and .foregroundColor(.secondary) for the status text to keep it unobtrusive.
  </action>
  <verify>Build succeeds. Document rows show appropriate progress indicators based on pipeline status. Status labels are human-readable.</verify>
  <done>In-pipeline documents show spinner + status label. Completed documents show checkmark badge. Status updates animate smoothly as polling updates arrive.</done>
