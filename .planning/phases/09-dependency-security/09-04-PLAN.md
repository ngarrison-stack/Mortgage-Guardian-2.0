# 09-04: Audit Verification and Cleanup

<objective>
Run final `npm audit`, verify vulnerability count is at or near zero, clean up root package.json, and document remaining accepted risks.
</objective>

## Tasks

### 1. Run `npm audit` across all workspaces

```bash
cd backend-express && npm audit
cd frontend && npm audit
```

### 2. Address remaining vulnerabilities

For each remaining alert:
- If fixable with `npm audit fix` — apply
- If requires `--force` — evaluate individually
- If unfixable (deep transitive dep) — document as accepted risk

### 3. Clean up root-level package.json

The root `package.json` has only `{ "dependencies": { "plaid": "^39.1.0" } }` which duplicates backend-express. Either remove it or convert to a workspace root.

### 4. Clean up Mortgage-Guardian-2.0 submodule

Same issue — duplicate `plaid` dependency in submodule.

### 5. Document accepted risks

Create `.planning/SECURITY-NOTES.md` listing:
- Any vulnerabilities that can't be fixed without major rewrites
- Mitigation strategies for each (e.g., "path-to-regexp ReDoS mitigated by Joi input validation")
- Review schedule

## Verification

- `npm audit` shows 0 critical, 0 high (or documented exceptions)
- All tests pass
- No orphaned lock files

- `package.json` (root) — cleaned up
