# 09-03: Evaluate Major Version Upgrades

<objective>
Assess and selectively apply major version upgrades that fix critical/high vulnerabilities, with careful testing for breaking changes.
</objective>

## Context

These packages have major version updates available:

### Backend — Major upgrades

| Package | Current | Latest | Risk | Notes |
|---------|---------|--------|------|-------|
| `express` | 4.x | 5.x | HIGH | Major rewrite — `path-to-regexp` ReDoS fix, but many breaking changes |
| `multer` | 1.x | 2.x | LOW | Removing in 09-01 — skip |
| `plaid` | 39.x | 41.x | MEDIUM | API version bump, may change method signatures |
| `rate-limiter-flexible` | 8.x | 9.x | LOW | Removing in 09-01 — skip |
| `@anthropic-ai/sdk` | 0.68 | 0.78 | MEDIUM | Pre-1.0, breaking changes possible |
| `file-type` | 16.x | 21.x | SKIP | v17+ is ESM-only, CJS not supported (known decision from Phase 4) |

### Frontend — Major upgrades

| Package | Current | Latest | Risk | Notes |
|---------|---------|--------|------|-------|
| `next` | 15.x | 16.x | HIGH | Major framework upgrade |
| `react` | 19.1 | 19.2 | LOW | Minor within major |
| `eslint` | 9.x | 10.x | MEDIUM | Config format changes |

## Tasks

### 1. Express 5.x evaluation

Express 5 is a significant upgrade. Key breaking changes:
- `path-to-regexp` v8 (fixes ReDoS — this is the critical vuln)
- Removed `app.del()`, changed `req.host`, `res.json()` behavior
- Changed default error handling

**Decision needed**: Upgrade to Express 5, or stay on 4.x and accept the `path-to-regexp` risk (mitigated by Joi validation on all inputs).

### 2. Plaid SDK update (39 → 41)

Check Plaid changelog for breaking changes in our used methods:
- `linkTokenCreate`, `itemPublicTokenExchange`, `accountsGet`, `transactionsGet`
- `itemGet`, `itemWebhookUpdate`, `itemRemove`
- `sandboxPublicTokenCreate`, `categoriesGet`

### 3. Anthropic SDK update (0.68 → 0.78)

Check for breaking changes in `messages.create()` which is our only API call.

### 4. Frontend: React 19.2 + Next.js (hold on 16.x)

- React 19.1 → 19.2: Safe minor bump, do it
- Next.js 15 → 16: Too risky for a security patch — defer

### 5. Run full test suite after each upgrade

Each major upgrade should be tested independently.

## Verification

- `npm test` — 488+ tests pass after each upgrade
- `npm audit` — critical/high count reduced significantly
- No runtime errors in development mode
