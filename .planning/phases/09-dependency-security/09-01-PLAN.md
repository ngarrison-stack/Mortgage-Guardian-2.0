# 09-01: Remove Unused Dependencies

<objective>
Remove phantom dependencies that are listed in package.json but not imported by any production code. This eliminates vulnerabilities from packages that aren't even used.
</objective>

## Context

Audit found these packages in backend-express/package.json that have zero or aspirational-only usage:

| Package | In package.json | Actually used? | Notes |
|---------|----------------|----------------|-------|
| `aws-sdk` | Not listed (transitive) | `financialSecurity/config.js` only | Security service is aspirational — AWS KMS/HSM/SecretsManager not deployed |
| `winston-elasticsearch` | Not listed (transitive) | `financialSecurity/config.js` only | Elasticsearch transport not deployed |
| `speakeasy` | Listed | `financialSecurity/helpers.js` only | MFA/TOTP — aspirational, not deployed |
| `multer` | Listed | **ZERO imports** | File upload uses base64 in request body, not multipart |
| `winston-syslog` | Listed | **ZERO imports** | Never imported anywhere |
| `rate-limiter-flexible` | Listed | `financialSecurity/config.js` only | App uses `express-rate-limit` instead |

## Tasks

### 1. Remove completely unused packages

```bash
npm uninstall multer winston-syslog
```

These have ZERO imports anywhere in the codebase.

### 2. Remove aspirational-only packages from dependencies

The `financialSecurity/` modules import `aws-sdk`, `winston-elasticsearch`, `speakeasy`, and `rate-limiter-flexible` — but these are for future AWS/HSM deployment that isn't happening yet. The service is fully mocked in tests.

Options:
- **Option A**: Remove from package.json, add try/catch optional requires in config.js (graceful degradation)
- **Option B**: Move to a separate `optionalDependencies` section
- **Option C**: Leave them but document as aspirational

**Recommended: Option A** — removes the vulnerability surface now, service already handles missing AWS gracefully

### 3. Verify no breakage

- `npm test` — 488 tests pass
- `grep -r "require('multer')" --include="*.js"` — 0 matches
- Startup still works
