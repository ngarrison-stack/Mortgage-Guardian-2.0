Create `ENV-GUIDE.md` at the repo root documenting every environment variable across all services:

Structure:
1. **Quick Start** — minimum vars needed to run locally (copy-paste ready)
2. **Backend Variables** — table with: Variable | Required | Default | Description | Source/Dashboard | Rotation
3. **Frontend Variables** — same table format
4. **Environment Matrix** — which vars are needed per environment (local dev, CI, staging, production)
5. **Secrets Classification**:
   - Critical (compromise = data breach): SUPABASE_SERVICE_KEY, DOCUMENT_ENCRYPTION_KEY, ANTHROPIC_API_KEY
   - High (compromise = financial exposure): PLAID_SECRET, PLAID_WEBHOOK_VERIFICATION_KEY, CLERK_SECRET_KEY

   - Low (non-secret config): PORT, NODE_ENV, LOG_LEVEL, ALLOWED_ORIGINS, feature flags
6. **Rotation Strategy** — per classification level:
   - Critical: Rotate every 90 days or on any suspected compromise
   - High: Rotate every 180 days
   - Medium: Rotate annually
   - How to rotate each key (dashboard URL or CLI command)
7. **Deployment Platform Reference** — where to set vars for Railway, Vercel, Supabase dashboard

Do NOT include actual secret values. Use placeholder format consistently (e.g., `sk-ant-your-key-here`).
