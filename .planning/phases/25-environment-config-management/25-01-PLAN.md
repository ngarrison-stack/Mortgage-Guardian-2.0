<task type="auto">
  <name>Task 1: Create comprehensive env validation utility</name>
  <files>backend-express/utils/envValidator.js, backend-express/utils/envValidator.test.js</files>
  <action>
Create `backend-express/utils/envValidator.js` that:

1. Defines all environment variables used across the codebase in a structured schema:
   - **Required** (server won't start without): SUPABASE_URL, SUPABASE_ANON_KEY, SUPABASE_SERVICE_KEY, DOCUMENT_ENCRYPTION_KEY
   - **Required for features** (warn but don't crash — features degrade gracefully): ANTHROPIC_API_KEY (AI analysis), PLAID_CLIENT_ID + PLAID_SECRET (banking), PLAID_ENV
   - **Optional with defaults**: PORT (3000), NODE_ENV (development), RATE_LIMIT_WINDOW_MS (900000), RATE_LIMIT_MAX_REQUESTS (100), ALLOWED_ORIGINS (*), LOG_LEVEL (info/debug based on NODE_ENV), PLAID_WEBHOOK_URL, PLAID_WEBHOOK_VERIFICATION_KEY
   - **Production-only required** (required when NODE_ENV=production): ALLOWED_ORIGINS (must not be *), PLAID_WEBHOOK_VERIFICATION_KEY
   - **Infrastructure optional**: REDIS_HOST, REDIS_PORT, REDIS_PASSWORD, JWT_SECRET, AWS_REGION, USE_CLOUD_HSM, VAULT_TOKEN, KMS_KEY_ID, KMS_SIGNING_KEY_ID, ELASTICSEARCH_URL, ELASTICSEARCH_USER, ELASTICSEARCH_PASSWORD

2. Export `validateEnvironment()` function that:
   - Skips validation when NODE_ENV=test
   - Checks required vars exist and are non-empty
   - Checks production-only vars when NODE_ENV=production
   - Logs warnings for missing feature vars (with which features are affected)
   - Throws descriptive error for missing required vars
   - Validates format where applicable: SUPABASE_URL must start with https://, DOCUMENT_ENCRYPTION_KEY must be 64 hex chars (32 bytes), PORT must be numeric
   - Returns a frozen config object with parsed values and defaults applied

3. Export `getConfig()` that returns the validated config singleton (call validateEnvironment() lazily on first access).

Use Joi for the validation schema since it's already a project dependency — this keeps patterns consistent. Do NOT add new dependencies.

Create `backend-express/__tests__/utils/envValidator.test.js` with tests covering:
- Missing required vars throws
- Missing feature vars warns but succeeds
- Production mode checks (ALLOWED_ORIGINS not *, webhook key required)
- Format validation (SUPABASE_URL format, DOCUMENT_ENCRYPTION_KEY hex length, PORT numeric)
- Default value application
- Test environment skips validation
  </action>
  <verify>cd backend-express && npx jest __tests__/utils/envValidator.test.js --no-cache --verbose</verify>
  <done>envValidator.js exports validateEnvironment() and getConfig(), all tests pass, no existing tests broken</done>
</task>
