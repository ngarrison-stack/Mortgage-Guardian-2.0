  <files>backend-express/schemas/consolidatedReportSchema.js, backend-express/__tests__/services/reportAggregationService.test.js</files>
  <action>
In `consolidatedReportSchema.js`, the `confidenceBreakdownSchema` requires all three scores to be non-null numbers:
```
documentAnalysis: Joi.number().min(0).max(100).required(),
forensicAnalysis: Joi.number().min(0).max(100).required(),
complianceAnalysis: Joi.number().min(0).max(100).required()
```

But `confidenceScoringService.calculateConfidence()` returns `null` for forensicAnalysis and complianceAnalysis when those upstream reports are missing. This causes schema validation to fail on every report with partial data.

Fix: Change `forensicAnalysis` and `complianceAnalysis` in `confidenceBreakdownSchema` to allow null:
```
forensicAnalysis: Joi.number().min(0).max(100).allow(null).required(),
complianceAnalysis: Joi.number().min(0).max(100).allow(null).required(),
```

Keep `documentAnalysis` as non-nullable (documents are always present for a valid case).

Also allow the optional `classificationImpact` field on `confidenceScoreSchema` to pass through validation (it was added in Phase 20-05 but not reflected in the Joi schema). Add:
```
classificationImpact: Joi.object({
  confidenceUsed: Joi.number().min(0).max(1).required(),
  factor: Joi.number().required(),
  layerAffected: Joi.string().required()
}).optional()
```

Update or add a test in the report aggregation test suite verifying that `validateConsolidatedReport()` passes when forensic/compliance breakdown values are null.
