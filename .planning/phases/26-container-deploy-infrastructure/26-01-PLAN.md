<task type="auto">
  <name>Task 1: Create backend production Dockerfile and .dockerignore</name>
  <files>backend-express/Dockerfile, backend-express/.dockerignore</files>
  <action>
1. Create `backend-express/.dockerignore`:
   - node_modules, .env*, __tests__, *.test.js, .git, coverage, .planning, *.md (except package*.json)

2. Create `backend-express/Dockerfile` with multi-stage build:
   - **Stage 1 (deps):** `FROM node:20-alpine AS deps`
     - WORKDIR /app
     - Copy package.json and package-lock.json
     - `RUN npm ci --omit=dev` (production deps only)
   - **Stage 2 (runtime):** `FROM node:20-alpine`
     - Create non-root user: `RUN addgroup -g 1001 -S appgroup && adduser -S appuser -u 1001 -G appgroup`
     - WORKDIR /app
     - Copy node_modules from deps stage
     - Copy application source (server.js, routes/, services/, middleware/, schemas/, utils/)
     - Set NODE_ENV=production
     - EXPOSE 3000
     - HEALTHCHECK: `CMD wget --no-verbose --tries=1 --spider http://localhost:3000/health || exit 1` (interval 30s, timeout 5s, retries 3)
     - USER appuser
     - CMD ["node", "server.js"]

   Do NOT use `npm start` in CMD — use `node server.js` directly to avoid unnecessary npm process wrapper.
   Do NOT install dev dependencies in production image.
   Do NOT copy __tests__ or test files into the image.
  </action>
  <verify>cd backend-express && docker build -t mg-backend:test . 2>&1 | tail -5 && docker image ls mg-backend:test --format "{{.Size}}"</verify>
  <done>Dockerfile builds successfully, image size under 200MB, uses non-root user, health check configured</done>
</task>

<task type="auto">
  <name>Task 2: Create frontend production Dockerfile and .dockerignore</name>
  <files>frontend/Dockerfile, frontend/.dockerignore, frontend/next.config.ts</files>
  <action>
1. Create `frontend/.dockerignore`:
   - node_modules, .env*, .next, .git, coverage, .planning, *.md

2. Modify `frontend/next.config.ts` to add `output: "standalone"` to the Next.js config. This is required for Docker — it creates a self-contained build that includes only necessary node_modules. Do NOT remove or modify any existing config (especially the rewrites).

3. Create `frontend/Dockerfile` with multi-stage build:
   - **Stage 1 (deps):** `FROM node:20-alpine AS deps`
     - WORKDIR /app
     - Copy package.json and package-lock.json
     - `RUN npm ci` (need all deps for build including devDependencies)
   - **Stage 2 (builder):** `FROM node:20-alpine AS builder`
     - WORKDIR /app
     - Copy node_modules from deps
     - Copy all source files
     - Set build-time env vars with ARG/ENV for NEXT_PUBLIC_ vars (use placeholder defaults so build succeeds without real values):
       - NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_placeholder
       - NEXT_PUBLIC_API_URL=http://localhost:3000
       - NEXT_PUBLIC_APP_URL=http://localhost:3001
       - NEXT_PUBLIC_APP_NAME="Mortgage Guardian"
       - NEXT_PUBLIC_ENABLE_PLAID=false
       - NEXT_PUBLIC_ENABLE_AI_ANALYSIS=false
     - Also set CLERK_SECRET_KEY=sk_test_placeholder for build
     - `RUN npm run build`
   - **Stage 3 (runtime):** `FROM node:20-alpine`
     - Create non-root user (same pattern as backend)
     - WORKDIR /app
     - Copy from builder: `.next/standalone/`, `.next/static/`, `public/` (if exists)
     - Set NODE_ENV=production, HOSTNAME="0.0.0.0", PORT=3001
     - EXPOSE 3001
     - USER appuser
     - CMD ["node", "server.js"] (Next.js standalone server)

   The standalone output produces a server.js at `.next/standalone/server.js` that serves the app.
   NEXT_PUBLIC_ vars are baked in at build time — real values must be provided as build args for production builds.
  </action>
  <verify>cd frontend && docker build -t mg-frontend:test . 2>&1 | tail -5 && docker image ls mg-frontend:test --format "{{.Size}}"</verify>
  <done>Dockerfile builds successfully, image size under 300MB, standalone output enabled, uses non-root user</done>
</task>
