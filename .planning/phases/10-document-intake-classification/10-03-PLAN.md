**2. ClassificationService class with methods:**

- `classifyDocument(documentText, { existingType } = {})` — Main entry point. Sends text to Claude with classification prompt. Returns:
  ```javascript
  {
    classificationType: 'servicing',       // broad category
    classificationSubtype: 'escrow_analysis', // specific type
    confidence: 0.92,                      // 0-1
    extractedMetadata: {                   // key fields found in doc
      dates: ['2024-01-15', '2024-02-15'],
      amounts: ['$1,245.67', '$892.30'],
      parties: ['Wells Fargo Home Mortgage', 'John Smith'],
      accountNumbers: ['****1234'],
      propertyAddress: '123 Main St, Springfield, IL 62701'
    },
    reasoning: 'Document contains escrow account analysis...'
  }
  ```

- `_buildClassificationPrompt(documentText, existingType)` — Build the Claude prompt. Key elements:
  - System context: "You are a forensic mortgage document classifier working in a litigation support role."
  - Provide the full DOCUMENT_TAXONOMY as reference
  - Ask Claude to: (1) classify into type/subtype, (2) extract key metadata, (3) provide confidence score, (4) explain reasoning
  - If existingType provided, tell Claude the user's initial classification for context but still classify independently
  - Require JSON response format matching the return schema above
  - Include instruction: "If the document doesn't fit any category, use classificationType: 'unknown' with subtype: 'unclassified'"

- `_parseClassificationResponse(responseText)` — Parse Claude's JSON response. Handle malformed JSON gracefully (wrap in { rawResponse, parseError } if parsing fails). Validate that classificationType exists in DOCUMENT_TAXONOMY (or is 'unknown'). Clamp confidence to 0-1.

- `getValidTypes()` — Return the DOCUMENT_TAXONOMY constant (for API consumers to know valid types)

- `getSubtypes(classificationType)` — Return subtypes array for a given type

**Important:**
- Use Claude model 'claude-sonnet-4-5-20250514' (fast, cost-effective for classification — not the full opus model)
- maxTokens: 2048 (classification responses are structured and concise)
- temperature: 0.1 (deterministic classification)
- Initialize Anthropic client from env var (same pattern as claudeService.js)
- Logger: `createLogger('classification')`
- Export taxonomy: `module.exports.DOCUMENT_TAXONOMY = DOCUMENT_TAXONOMY;`
