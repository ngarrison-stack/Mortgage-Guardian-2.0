**1. Create migration for pipeline state persistence:**

Create `003_pipeline_state.sql`:
```sql
-- Pipeline state table for tracking document processing progress
CREATE TABLE IF NOT EXISTS pipeline_state (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  document_id text NOT NULL,
  user_id text NOT NULL,
  document_type text,
  status text NOT NULL DEFAULT 'uploaded',
  steps jsonb DEFAULT '{}',
  extracted_text text,
  classification_results jsonb,
  analysis_results jsonb,
  case_id uuid REFERENCES case_files(id) ON DELETE SET NULL,
  error jsonb,
  retry_count integer DEFAULT 0,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_pipeline_state_document ON pipeline_state(document_id);
CREATE INDEX IF NOT EXISTS idx_pipeline_state_user_status ON pipeline_state(user_id, status);
```

**2. Add DB persistence to documentPipelineService.js:**

Add a Supabase client initialization (same pattern as documentService.js).

Add private methods:
- `_persistPipeline(pipeline)` — Upsert pipeline state to `pipeline_state` table. Use `document_id` as the natural key. If Supabase not configured, skip (in-memory only). Do NOT throw on DB errors — log warning and continue (pipeline operation should not fail because of state persistence).
- `_loadPipeline(documentId)` — Load from `pipeline_state` table first, fall back to in-memory Map. This enables server-restart resilience.

Update existing methods:
- `initPipeline`: After creating in-memory state, call `_persistPipeline`
- `_advanceState`: After updating state, call `_persistPipeline`
- `processDocument`: At start, try `_loadPipeline` before checking in-memory Map
- `getStatus`: Try `_loadPipeline` as fallback if not in Map

**3. Add case file auto-association:**

Add method `_associateWithCase(pipeline)`:
- Called after classification completes (in processDocument, after _runClassification)
- Use caseFileService to check if user has an open case
- If exactly one open case exists, auto-associate: call `caseFileService.addDocumentToCase({ caseId, documentId, userId })`
- If no open cases, skip (user can manually associate later)
- If multiple open cases, skip (ambiguous — user decides)
- Store caseId on pipeline state
- Log the association or skip reason

Import caseFileService at the top.

**Important:**
- DB persistence is best-effort — never block pipeline on DB write failures
- The in-memory Map remains the primary source for active pipelines (fast)
- DB is the recovery mechanism for server restarts
- _loadPipeline should populate the in-memory Map when loading from DB
