**1. Create `schemas/cases.js`:**

Joi schemas for all case file endpoints:
- `createCaseSchema`: body — caseName (required, string, 1-200 chars), borrowerName (optional string), propertyAddress (optional string), loanNumber (optional string), servicerName (optional string), notes (optional string)
- `getCasesSchema`: query — status (optional, valid: 'open', 'in_review', 'complete', 'archived'), limit (optional int 1-500 default 50), offset (optional int min 0 default 0)
- `getCaseSchema`: no params needed (caseId from URL, userId from auth)
- `updateCaseSchema`: body — caseName (optional), borrowerName (optional), propertyAddress (optional), loanNumber (optional), servicerName (optional), status (optional, valid values), notes (optional). At least one field required (Joi.object().min(1))
- `addDocumentToCaseSchema`: body — documentId (required string)
- `removeDocumentFromCaseSchema`: no body needed (documentId from URL param)

**2. Create `routes/cases.js`:**

Express router with these endpoints:
- `POST /v1/cases` — Create a new case. Extract userId from req.user.id (set by auth middleware). Call caseFileService.createCase(). Return 201.
- `GET /v1/cases` — List user's cases. Extract userId from req.user.id. Optional status filter from query. Call caseFileService.getCasesByUser(). Return 200.
- `GET /v1/cases/:caseId` — Get case with its documents. Extract userId from req.user.id. Call caseFileService.getCase(). Return 404 if not found.
- `PUT /v1/cases/:caseId` — Update case. Extract userId from req.user.id. Call caseFileService.updateCase(). Return 404 if not found.
- `DELETE /v1/cases/:caseId` — Delete case. Extract userId from req.user.id. Call caseFileService.deleteCase().
- `POST /v1/cases/:caseId/documents` — Add document to case. Call caseFileService.addDocumentToCase().
- `DELETE /v1/cases/:caseId/documents/:documentId` — Remove document from case. Call caseFileService.removeDocumentFromCase().

**Important implementation notes:**
- Use `req.user.id` for userId (from JWT auth middleware) — NOT from request body. This is more secure than the v2.0 pattern of passing userId in the request body.
- Apply `validate()` middleware on each route
- Use `createLogger('case-routes')` for structured logging
- Follow existing error handling pattern: try/catch, logger.error, next(error)
- If caseFileService returns null for getCase, return 404

**3. Register routes in `server.js`:**
- Import: `const caseRoutes = require('./routes/cases');`
- Mount: `app.use('/v1/cases', caseRoutes);`
- Add case routes to the 404 handler's availableRoutes list
