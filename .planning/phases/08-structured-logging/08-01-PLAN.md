# 08-01: Winston Logger Configuration

<objective>
Create a shared Winston logger utility (`utils/logger.js`) with environment-aware transports, structured JSON format, child logger factory, and Morgan integration — serverless-compatible for Vercel/Railway.
</objective>

## Context

- Winston v3.11.0 already in dependencies
- `financialSecurity/config.js` has a domain-specific logger (file + Elasticsearch transports) — NOT suitable as the shared app logger
- App deploys to Vercel (serverless) and Railway — both capture stdout, file transports won't work on Vercel
- Morgan already used for HTTP request logging in server.js
- 119 console.* statements across 10 files need structured replacement (subsequent plans)

## Tasks

### 1. Create `utils/logger.js`

Shared Winston logger with:
- **Levels**: Standard Winston (error, warn, info, http, verbose, debug, silly)
- **Production format**: JSON with timestamp, service name, log level
- **Development format**: Colorized, human-readable with timestamp
- **Transport**: Console only (stdout) — serverless-friendly
- **LOG_LEVEL env var**: Control verbosity per environment (default: 'info' in prod, 'debug' in dev)
- **Child logger factory**: `createLogger(serviceName)` returns `logger.child({ service: serviceName })`
- **Sensitive data**: Log level for the base logger, child loggers inherit

### 2. Create Morgan-Winston stream

- Export a `morganStream` object with a `write(message)` method that calls `logger.http(message.trim())`
- This replaces Morgan's default stdout with Winston's HTTP log level

### 3. Wire Morgan into Winston in `server.js`

- Replace `app.use(morgan('dev'))` / `app.use(morgan('combined'))` with `app.use(morgan('combined', { stream: morganStream }))`
- Remove the dev/prod branching — 'combined' format for both, Winston handles formatting

### 4. Replace server.js console.* statements

- Startup banner: `logger.info('Server started', { port, env, services: { claude, plaid, supabase } })`
- Error handler: `logger.error('Unhandled error', { error: err.message, stack: err.stack, path: req.path })`
- SIGTERM: `logger.info('SIGTERM received, shutting down')`
