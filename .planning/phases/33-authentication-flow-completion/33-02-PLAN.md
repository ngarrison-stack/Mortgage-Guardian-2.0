**Current state:**
- AuthManager: basic Clerk sign-in/sign-up, `updateAPIToken()` fetches JWT once after auth
- APIClient: throws `APIError.authenticationRequired` on 401 but nothing handles it
- No session persistence — user must re-login on every app launch
- No token refresh — if Clerk JWT expires mid-session, API calls fail silently
- Email verification methods are commented out (Clerk SDK API may have changed)

**Clerk iOS SDK patterns:**
- `Clerk.shared.session?.getToken()` returns fresh JWT (Clerk handles refresh internally)
- `Clerk.shared.user` is nil until `Clerk.shared.configure()` completes loading
- Clerk persists session in Keychain automatically — `Clerk.shared.user` should be available after restart IF configure() is called
- The Clerk SDK `configure()` is async — need to wait for it before checking auth state

**Constraining decisions:**
- Clerk iOS SDK v0.71.4 is installed
- APIClient is @Observable with setAuthToken() method (from Phase 32)
- App uses @StateObject for AuthManager in MortgageGuardianApp.swift

<task type="auto">
  <name>Task 1: Add token lifecycle and session persistence to AuthManager</name>
  <files>MortgageGuardian/Services/AuthManager.swift</files>
  <action>
Rewrite AuthManager to handle the full token lifecycle:

1. **Async initialization with Clerk session restoration:**
   - Add `@Published var isLoading = true` for splash screen state
   - In `setupClerk()`, after `Clerk.shared.configure()`, check `Clerk.shared.user` to restore session
   - Set `isLoading = false` after session check completes
   - If user exists, call `refreshToken()` to set fresh JWT on APIClient

2. **Token refresh method:**
   - Add `func refreshToken() async` that calls `Clerk.shared.session?.getToken()`
   - On success: `APIClient.shared.setAuthToken(token.jwt)`
   - On failure: call `signOut()` (token revoked or session expired)
   - This is the single source of truth for getting fresh tokens

3. **Periodic token refresh:**
   - Add a `tokenRefreshTask: Task<Void, Never>?` property
   - After successful auth, start a background Task that refreshes every 4 minutes (Clerk JWTs expire after 5 min by default)
   - Cancel the task on sign-out
   - Use `Task.sleep(for: .seconds(240))` between refreshes

4. **Clean up sign-in/sign-up/signOut:**
   - `signIn()`: After successful sign-in, call `refreshToken()` then start periodic refresh
   - `signUp()`: Leave as-is (sign-up → verify → sign-in flow)
   - `signOut()`: Cancel refresh task, clear API token, reset state
   - Remove commented-out email verification code — it will be addressed when the full sign-up flow is needed

5. **Public `handleAuthenticationRequired()` method:**
   - Called when APIClient gets 401
   - Attempts one token refresh
   - If refresh fails, signs out user (triggers UI navigation to LoginView)

Do NOT add Keychain storage — Clerk SDK handles session persistence internally.

**APIClient.swift changes:**

1. Add a `var onAuthenticationRequired: (() async -> Void)?` callback property
2. In `performRequest()`, when a 401 is received:
   - Call `onAuthenticationRequired?()` before throwing
   - This allows AuthManager to attempt token refresh
3. Keep throwing `APIError.authenticationRequired` after the callback (callers still need to handle it)

**MortgageGuardianApp.swift changes:**

1. Wire up the auth callback on app launch:
   ```swift
   APIClient.shared.onAuthenticationRequired = { [weak authManager] in
       await authManager?.handleAuthenticationRequired()
   ```

2. Show a loading/splash state while `authManager.isLoading` is true:
   - If `authManager.isLoading`: show a simple ProgressView or app icon splash
   - If `!authManager.isLoading && authManager.isSignedIn`: show MainTabView
   - If `!authManager.isLoading && !authManager.isSignedIn`: show LoginView

3. Remove the `setupCrashProtection()` method call if it references undefined `plaidLinkService` — it causes a build error since that variable isn't in scope.

Do NOT change the public `request()` signature — callers remain unchanged.
