**Auth architecture:**
- iOS app: Clerk iOS SDK v0.71.4 → `session.getToken()` → JWT → `APIClient.setAuthToken()`
- Express backend: `auth.js` middleware validates `Authorization: Bearer <token>` via `supabase.auth.getUser(token)`
- The `jsonwebtoken` package is already installed in backend-express
- Express has `JWT_SECRET` env var used by `financialSecurity/validation.js`

**Clerk JWT structure:**
- Clerk JWTs are standard RS256 JWTs
- Issuer: `https://<clerk-domain>` (from publishable key — `fond-mako-71.clerk.accounts.dev`)
- JWKS URL: `https://fond-mako-71.clerk.accounts.dev/.well-known/jwks.json`
- Contains `sub` (user ID), `iss`, `exp`, `iat`, `azp` claims

**Constraining decisions:**
- Clerk authentication must work with Express backend JWT tokens (v6.0 constraint)
- Backend must continue accepting Supabase tokens for web frontend (Clerk Next.js frontend uses different auth path)
- Keep existing test suite passing (1636+ tests)

<task type="auto">
  <name>Task 1: Add Clerk JWT verification to auth middleware</name>
  <files>backend-express/middleware/auth.js, backend-express/package.json</files>
  <action>
Install `jwks-rsa` package for fetching Clerk's JWKS public keys:
```
cd backend-express && npm install jwks-rsa
```

Rewrite `auth.js` to try two verification paths in sequence:

1. **Try Clerk JWKS verification first** (for iOS/mobile tokens):
   - Use `jwks-rsa` to create a JWKS client pointing at `https://${CLERK_DOMAIN}/.well-known/jwks.json`
   - Get the `CLERK_DOMAIN` from env var `CLERK_ISSUER_URL` (default: `https://fond-mako-71.clerk.accounts.dev`)
   - Verify JWT signature using RS256 and the signing key from JWKS
   - Verify standard claims: `iss` matches Clerk domain, `exp` not expired
   - On success: attach `req.user = { id: decoded.sub, email: decoded.email, provider: 'clerk' }` and call `next()`

2. **Fall back to Supabase validation** (for web frontend tokens):
   - If Clerk verification fails (wrong issuer, invalid signature), try existing `supabase.auth.getUser(token)`
   - On success: attach `req.user` as before with `provider: 'supabase'` added
   - On failure: return 401

3. **If both fail:** return 401 with `{ error: 'Unauthorized', message: 'Invalid or expired token' }`

Keep the `PUBLIC_PATHS` and `isPublicPath()` logic unchanged.

Add `CLERK_ISSUER_URL` to `.env.example` with comment explaining it.

Do NOT use `jsonwebtoken` for this — use `jwks-rsa` with its built-in JWT verification. The `jwks-rsa` library handles key rotation automatically.

Cache the JWKS signing key (jwks-rsa does this by default with `cache: true, rateLimit: true`).
