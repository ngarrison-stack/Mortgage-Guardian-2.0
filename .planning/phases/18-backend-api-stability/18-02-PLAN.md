**Bug 1 — CORS credentials/origin incompatibility (server.js:38-43):**
- `credentials: true` with `origin: '*'` violates CORS spec
- Browsers silently reject credentialed requests when origin is wildcard
- Impact: JWT auth fails silently in production if ALLOWED_ORIGINS='*'
- Fix: When credentials is true and origin is '*', use a function that echoes the request origin

**Bug 2 — Webhook signature verification silent fallback (plaidService.js:~468-497):**
- If PLAID_WEBHOOK_VERIFICATION_KEY env var is missing, verification returns true (skips check)
- In production, this allows any forged webhook to manipulate transaction data
- Impact: Security vulnerability — unauthenticated webhook injection
- Fix: In production (NODE_ENV=production), throw error if key is missing instead of returning true

**Established patterns:**
- Winston structured logging with child loggers
- Environment-aware behavior (NODE_ENV checks)
