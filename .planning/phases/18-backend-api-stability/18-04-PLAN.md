# Key files:
@backend-express/routes/documents.js
@backend-express/middleware/auth.js

**Bug 1 — User ID isolation bypass (documents.js:~161):**
- GET `/v1/documents/:documentId` takes `userId` from `req.query` instead of `req.user.id`
- Authenticated user can access other users' documents by passing `?userId=OTHER_USER_ID`
- This is an authorization bypass — IDOR (Insecure Direct Object Reference) vulnerability
- Fix: Use `req.user.id` from JWT auth middleware for all user-scoped operations

**Bug 2 — Error returned with 200 status (documents.js:136-142):**
- When analysis fails, returns `res.status(200).json({ status: 'error', ... })`
- Client code checking `res.status` or `res.ok` will treat errors as success
- Fix: Return 422 (Unprocessable Entity) for analysis failures

**Established patterns:**
- `requireAuth` middleware sets `req.user` with user details from JWT
- Joi validation at API boundary via `validate()` middleware
- Routes use `req.user.id` for authenticated user context (other routes do this correctly)
