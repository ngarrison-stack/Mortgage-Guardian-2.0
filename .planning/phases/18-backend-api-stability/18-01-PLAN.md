<task type="auto">
  <name>Task 1: Implement graceful shutdown with connection cleanup</name>
  <files>backend-express/server.js</files>
  <action>Replace the existing SIGTERM handler (lines ~157-161) with a proper graceful shutdown function that:

1. Sets a `shuttingDown` flag to reject new requests (return 503)
2. Calls `server.close()` to stop accepting new connections and wait for in-flight requests
3. Logs shutdown initiation with Winston
4. Sets a 10-second timeout to force exit if cleanup hangs
5. Calls `process.exit(0)` after server closes

Add identical SIGINT handler that calls the same shutdown function.

Do NOT add Redis/Supabase disconnect calls — these are SDK clients that clean up on process exit. Focus on the HTTP server and in-flight requests only.

Avoid adding middleware to check shuttingDown flag — Express server.close() already handles this by not accepting new connections.</action>
  <verify>Start server with `cd backend-express && node server.js &`, then send SIGTERM with `kill $!`. Check logs show "SIGTERM received, starting graceful shutdown" and "Server closed, exiting". Verify process exits cleanly.</verify>
  <done>SIGTERM and SIGINT both trigger graceful shutdown. Server stops accepting new connections, waits for in-flight requests (up to 10s timeout), then exits with logged messages.</done>
</task>

<task type="auto">
  <name>Task 2: Add uncaught exception and unhandled rejection handlers</name>
  <files>backend-express/server.js</files>
  <action>Add process-level error handlers near the top of server.js (after logger initialization but before Express app setup):

1. `process.on('uncaughtException', (err) => { ... })` — Log error with full stack via Winston, then call process.exit(1). Do NOT try to continue — uncaught exceptions leave Node in an undefined state.

2. `process.on('unhandledRejection', (reason, promise) => { ... })` — Log warning with reason via Winston. Do NOT exit — unhandled rejections are recoverable. Log enough context to debug (stringify reason if it's not an Error).

Avoid swallowing errors or continuing after uncaughtException. The Node.js docs explicitly warn against this — the process state is undefined after an uncaught exception.</action>
  <verify>Add a temporary `setTimeout(() => { throw new Error('test uncaught'); }, 1000)` to server.js, start server, confirm Winston logs the error and process exits with code 1. Remove the test throw afterward.</verify>
  <done>uncaughtException handler logs error with stack trace and exits with code 1. unhandledRejection handler logs warning with reason. Both use Winston structured logging.</done>
</task>
