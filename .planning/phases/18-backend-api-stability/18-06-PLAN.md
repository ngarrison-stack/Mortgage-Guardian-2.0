1. Read `.env.example` to understand all available env vars, then define validation:

```javascript
function validateEnvironment() {
  const required = {
    // Core — server won't function without these
    'SUPABASE_URL': 'Supabase database connection',
    'SUPABASE_ANON_KEY': 'Supabase anonymous key for auth',
  };

  const recommended = {
    // Services — specific features break without these
    'ANTHROPIC_API_KEY': 'Claude AI document analysis',
    'PLAID_CLIENT_ID': 'Plaid banking integration',
    'PLAID_SECRET': 'Plaid banking integration',
  };

  const missing = [];
  const warnings = [];

  for (const [key, desc] of Object.entries(required)) {
    if (!process.env[key]) missing.push(`${key} (${desc})`);
  }

  for (const [key, desc] of Object.entries(recommended)) {
    if (!process.env[key]) warnings.push(`${key} (${desc})`);
  }

  if (warnings.length > 0) {
    logger.warn('Missing recommended environment variables', { missing: warnings });
  }

  if (missing.length > 0) {
    logger.error('Missing required environment variables', { missing });
    throw new Error(`Missing required environment variables: ${missing.join(', ')}`);
  }

  logger.info('Environment validation passed');
}
```

2. Call `validateEnvironment()` BEFORE `app.listen()` — if it throws, the server never starts.

3. Skip validation when NODE_ENV is 'test' — tests may not have all env vars configured.

Do NOT make ANTHROPIC_API_KEY or PLAID keys strictly required — they're recommended. The server should start for health checks even without external service keys. Only database connection (Supabase) is truly required.</action>
