  <action>Start the backend server locally and run the validation script:

  1. Start the backend in production mode:
     ```bash
     cd backend-express
     NODE_ENV=production PORT=3099 node server.js &
     SERVER_PID=$!
     ```
     Note: Use port 3099 to avoid conflicts. The server may log warnings about missing env vars — that's expected without full configuration.

  2. Run validation script:
     ```bash
     ./scripts/validate-deployment.sh http://localhost:3099
     ```

  3. Capture and document results

  4. Stop the server:
     ```bash
     kill $SERVER_PID
     ```

  **If the server fails to start** (missing required env vars), test what we can:
  - Validate the script works syntactically
  - Run with a minimal .env that satisfies required vars (use dummy values)
  - Document which checks pass/fail/skip

  **Fix any issues found** in the validation script. Do NOT modify application code to make validation pass — document real failures as findings.

  **If Docker is available:** Also try `docker compose up --build -d` and run validation against that. Stop with `docker compose down` when done.

  The goal is to VALIDATE, not to achieve 100% pass rate. A deployment dry run with documented "needs Supabase connection" warnings is perfectly valid.</action>
  <verify>Validation script runs and produces output, results documented</verify>
  <done>Deployment validation executed, results captured, any script bugs fixed</done>
