<task type="auto">
  <name>Task 1: Create requireAuth middleware module</name>
  <files>backend-express/middleware/auth.js</files>
  <action>
Create `backend-express/middleware/auth.js` exporting a `requireAuth` Express middleware function.

Implementation:
1. Import `createClient` from `@supabase/supabase-js`
2. Initialize a module-level Supabase client using `process.env.SUPABASE_URL` and `process.env.SUPABASE_ANON_KEY` (NOT the service key — anon key is correct for JWT validation via auth.getUser)
3. If Supabase is not configured (missing env vars), log a warning but don't crash — allow the middleware to be loaded (for testing with mocks)

The `requireAuth` function:
- Extract the Authorization header from the request
- If missing or doesn't start with `Bearer `, return 401:
  `{ error: 'Unauthorized', message: 'Missing or invalid Authorization header' }`
- Extract the token (everything after 'Bearer ')
- Call `supabase.auth.getUser(token)` to validate
- If error or no user returned, return 401:
  `{ error: 'Unauthorized', message: 'Invalid or expired token' }`
- On success, set `req.user = user` (the user object from Supabase) and call `next()`
- Wrap the entire validation in try/catch — any unexpected error returns 401:
  `{ error: 'Unauthorized', message: 'Token validation failed' }`

Define a `PUBLIC_PATHS` array for paths that bypass auth:
- `POST /v1/plaid/webhook` (uses its own signature verification)

Check `req.originalUrl` and `req.method` against PUBLIC_PATHS before performing JWT validation. If the path matches, call `next()` immediately.

Export: `module.exports = { requireAuth }`

Follow existing conventions:
- CommonJS require/module.exports
- 2-space indentation, single quotes
- Error format matches: `{ error: 'Type', message: '...' }`
- Match the lazy init pattern from documentService.js (supabase client as module-level variable)
