  <name>Task 1: Create auth middleware unit tests</name>
  <files>backend-express/__tests__/middleware/auth.test.js</files>
  <action>
Create `backend-express/__tests__/middleware/auth.test.js` — unit tests for the requireAuth middleware function in isolation.

Mock `@supabase/supabase-js` using jest.mock() to replace createClient with a function that returns a mock Supabase client. Use the `createMockSupabaseClient` from `__tests__/mocks/mockSupabaseClient.js` for the mock implementation.

Test structure:
```
describe('requireAuth middleware', () => {
  // Setup: mock Supabase, create mock req/res/next

  describe('valid authentication', () => {
    test('passes request with valid Bearer token')
    test('sets req.user with authenticated user data')
    test('calls next() exactly once')
  })

  describe('missing authentication', () => {
    test('returns 401 when Authorization header is missing')
    test('returns 401 when Authorization header is empty')
    test('returns error format: { error: "Unauthorized", message: "..." }')
    test('does not call next()')
  })

  describe('invalid authentication', () => {
    test('returns 401 for non-Bearer scheme (e.g., "Basic")')
    test('returns 401 when Supabase returns error')
    test('returns 401 when Supabase returns null user')
    test('returns 401 for malformed token (Bearer with empty string)')
  })

  describe('public path exclusion', () => {
    test('skips auth for POST /v1/plaid/webhook')
    test('does NOT skip auth for GET /v1/plaid/webhook')
    test('does NOT skip auth for POST /v1/plaid/link_token')
  })
})
```

For req/res/next mocking:
- `req`: plain object with `headers`, `originalUrl`, `method` properties
- `res`: object with `status` (returns self for chaining), `json` (jest.fn())
- `next`: jest.fn()

Pattern for mock Supabase in tests:
```javascript
const { createMockSupabaseClient } = require('../mocks/mockSupabaseClient');
const mockClient = createMockSupabaseClient();

jest.mock('@supabase/supabase-js', () => ({
  createClient: jest.fn(() => mockClient)
}));

// Then require the middleware AFTER mocking
const { requireAuth } = require('../../middleware/auth');
```

Use `beforeEach` to reset the mock client between tests: `mockClient.reset()`.

For the "valid token" test, configure the mock:
```javascript
mockClient.setResponse('auth', {
  data: { user: { id: 'user-123', email: 'test@example.com', role: 'authenticated' } },
  error: null
});
```

For the "invalid token" test:
```javascript
mockClient.setError('auth', { message: 'Invalid token', code: 'INVALID_TOKEN' });
```

  <action>
Create `backend-express/__tests__/routes/auth-integration.test.js` — integration tests using supertest to verify route-level protection.

These tests hit actual Express routes via supertest, verifying the full middleware chain (rate limiter → auth → route handler).

Mock setup:
1. Mock `@supabase/supabase-js` with createMockSupabaseClient (same as Task 1)
2. Mock the service modules to prevent real API calls:
   - `jest.mock('../../services/claudeService')`
   - `jest.mock('../../services/plaidService')`
   - `jest.mock('../../services/documentService')`
   Use the mock modules from `__tests__/mocks/` for Claude, or simple jest.fn() returns for others.
3. Use `setupTestApp()` from testUtils to get a fresh app instance
4. Use `generateTestJWT()` from testUtils for token generation

Note: Since the auth middleware uses Supabase (not raw JWT verification), the test JWT from testUtils won't be validated by Supabase auth.getUser(). Instead, configure the mock Supabase client to return success for any token, and test that the middleware correctly passes/blocks based on Supabase's response.

Test structure:
```
describe('Route-level authentication', () => {
  describe('protected routes reject unauthenticated requests', () => {
    test('POST /v1/ai/claude/analyze → 401 without auth')
    test('POST /v1/ai/claude/test → 401 without auth')
    test('POST /v1/plaid/link_token → 401 without auth')
    test('POST /v1/plaid/accounts → 401 without auth')
    test('POST /v1/plaid/transactions → 401 without auth')
    test('POST /v1/documents/upload → 401 without auth')
    test('GET /v1/documents → 401 without auth')
    test('GET /v1/documents/test-id → 401 without auth')
    test('DELETE /v1/documents/test-id → 401 without auth')
  })

  describe('protected routes accept authenticated requests', () => {
    test('POST /v1/ai/claude/analyze → passes auth with valid token (may fail validation, but not 401)')
  })

  describe('public routes remain accessible', () => {
    test('GET /health → 200 without auth')
    test('GET / → 200 without auth')
  })
})
```

For authenticated requests:
- Set Authorization header: `.set('Authorization', 'Bearer valid-test-token')`
- Configure mock Supabase to return success for that token
- The route handler may return 400 (bad request due to missing body fields) — that's fine, the key assertion is that it does NOT return 401

For unauthenticated requests:
- Don't set Authorization header
- Mock Supabase returns error
- Assert status is 401
- Assert body matches `{ error: 'Unauthorized', message: expect.any(String) }`

Use supertest's `.expect()` for assertions where possible, fall back to Jest `expect()` for complex assertions.
