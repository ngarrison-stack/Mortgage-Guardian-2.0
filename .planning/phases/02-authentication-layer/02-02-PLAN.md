- Any route handler files

1. helmet, compression, cors, body parsing, morgan (existing)
2. Rate limiter on /v1/ (existing)
3. **requireAuth on /v1/** (NEW)

<task type="auto">
  <name>Task 3: Verify Plaid webhook exclusion</name>
  <files>backend-express/middleware/auth.js</files>

Verify the Plaid webhook path bypasses JWT auth:

Use supertest or node -e to send a POST to /v1/plaid/webhook without an Authorization header. The request should NOT receive a 401 — it should proceed to the webhook handler (which will then validate the webhook signature).

Since the webhook handler expects `express.raw()` body format and valid signature, the request will likely fail with a different error (not 401). The key assertion is: status code is NOT 401.

Run:
```
node -e "
const request = require('supertest');
const app = require('./server');
(async () => {
  const res = await request(app)
    .post('/v1/plaid/webhook')
    .send({});
  console.log('Webhook (no auth):', res.status, '- should NOT be 401');
  console.log('Pass:', res.status !== 401);
})();
"
```

If webhook returns 401, the PUBLIC_PATHS exclusion in auth.js is not working — fix it. Common issues:
- Path matching: ensure `req.originalUrl` starts with the public path (use `.startsWith()`)
- Method matching: ensure POST method is checked
- Middleware order: ensure requireAuth runs AFTER the path is resolved

If this verification reveals issues, fix the middleware before completing this task.
