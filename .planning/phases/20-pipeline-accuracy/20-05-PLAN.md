<task type="auto">
  <name>Task 2: Wire classification confidence into scoring pipeline</name>
  <files>backend-express/services/confidenceScoringService.js, backend-express/services/consolidatedReportService.js, backend-express/__tests__/services/confidenceScoringService.test.js</files>
  <action>
1. In confidenceScoringService.js, update calculateOverallScore() to accept an optional classificationConfidence parameter (0.0-1.0):
   - If provided, apply as a scaling factor on the documentAnalysis layer score:
     documentAnalysisScore = rawDocumentAnalysisScore × classificationConfidenceFactor
   - classificationConfidenceFactor:
     - confidence >= 0.7: factor = 1.0 (high confidence, no penalty)
     - confidence 0.4-0.7: factor = 0.85 (medium confidence, 15% reduction)
     - confidence < 0.4: factor = 0.65 (low confidence, 35% reduction)
   - Rationale: If classification is uncertain, the type-specific analysis built on that classification is less trustworthy. We reduce only the document analysis layer (not forensic or compliance) because those layers don't depend on classification type.

2. In consolidatedReportService.js (or wherever the scoring service is called), pass the classification confidence from the document's pipeline results into calculateOverallScore(). Find where aggregatedData is assembled and include classificationConfidence from the document's classification results.

3. Add a `classificationImpact` field to the scoring response:
   - { confidenceUsed: 0.45, factor: 0.85, layerAffected: 'documentAnalysis' }
   - This makes the scoring transparent — consumers can see how classification confidence affected the score.

4. Add tests:
   - High classification confidence (0.9) → documentAnalysis score unchanged (factor 1.0)
   - Medium classification confidence (0.55) → documentAnalysis score reduced 15%
   - Low classification confidence (0.3) → documentAnalysis score reduced 35%
   - No classification confidence provided → factor defaults to 1.0 (backward compatible)
   - classificationImpact field present in scoring response
   - Overall score reflects the classification factor (e.g., borderline document with low classification confidence tips from MEDIUM to HIGH risk)
  </action>
  <verify>cd backend-express && npm test -- --testPathPattern="confidenceScoring|consolidatedReport" --verbose 2>&1 | tail -40</verify>
  <done>Classification confidence wired into scoring, factor applied to document analysis layer, classificationImpact field in response, backward compatible when confidence not provided, all tests pass</done>
