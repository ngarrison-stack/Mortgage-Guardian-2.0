<task type="auto">
  <name>Task 1: Add file validation to document upload route</name>
  <files>backend-express/routes/documents.js</files>
  <action>
    In the POST /upload handler, after Joi validates the body and before calling documentService.uploadDocument():

    1. Decode base64 content: `const fileBuffer = Buffer.from(content, 'base64')`
    2. Call validateFileContent(fileBuffer, fileName) from utils/fileValidation.js
    3. If validation fails, return 400 with { error: 'Validation Error', message: result.error }
    4. Sanitize fileName: `const safeFileName = sanitizeFileName(fileName)`
    5. Pass safeFileName (not raw fileName) to documentService.uploadDocument()

    Do NOT create a separate middleware for this — the validation needs the decoded buffer which is specific to this route's logic. Keep it in the route handler.

    Also update the route to use safeFileName in the response and logging.
  </action>
  <verify>

    - sanitizeFileName applied to fileName
    - 400 response on invalid file content
    - safeFileName used throughout (not raw fileName)
  </verify>

<task type="auto">
  <name>Task 2: Harden body parser limit and update upload Joi schema</name>
  <files>backend-express/server.js, backend-express/schemas/documents.js</files>
  <action>
    1. In server.js, reduce the global body limit from 50mb to 25mb:
       `app.use(express.json({ limit: '25mb' }))` — 25MB accommodates base64 overhead (~33% larger than binary) for a 20MB PDF limit.
       Also reduce urlencoded limit: `app.use(express.urlencoded({ extended: true, limit: '25mb' }))`

    2. In schemas/documents.js, add stricter constraints to uploadDocumentSchema:
       - fileName: Add `.max(255)` and `.pattern(/^[a-zA-Z0-9._\s()-]+$/, 'safe characters')` to reject filenames with path separators or dangerous characters at the Joi level (defense in depth — sanitizeFileName handles this too)
       - content: Add `.max(28000000)` (28MB base64 ≈ 21MB binary, just above the 20MB PDF limit)
       - documentType: Add `.valid('mortgage_statement', 'bank_statement', 'tax_document', 'correspondence', 'legal_document', 'unknown')` to restrict to known document types

    Keep existing schema fields and defaults. Only add constraints.
  </action>
  <verify>
    cd backend-express && node -e "
      const { uploadDocumentSchema } = require('./schemas/documents');
      // Test fileName rejects path traversal
      let r1 = uploadDocumentSchema.validate({ documentId: 'x', userId: 'u', fileName: '../etc/passwd', content: 'abc' });
      console.log('Path traversal rejected:', !!r1.error);
      // Test valid fileName passes
      let r2 = uploadDocumentSchema.validate({ documentId: 'x', userId: 'u', fileName: 'report.pdf', content: 'abc' });
      console.log('Valid name passes:', !r2.error);
      // Test oversized content rejected
      let r3 = uploadDocumentSchema.validate({ documentId: 'x', userId: 'u', fileName: 'test.pdf', content: 'x'.repeat(28000001) });
      console.log('Oversized rejected:', !!r3.error);
    "
  </verify>
  <done>Body limit reduced to 25mb, upload schema enforces fileName pattern (no path separators), content size max (28MB), and documentType whitelist</done>
</task>
