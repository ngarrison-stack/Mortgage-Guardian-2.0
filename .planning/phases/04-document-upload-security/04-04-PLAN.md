    1. **Valid uploads accepted** (3-4 tests):
       - Valid PDF (real PDF magic bytes as base64) → 201
       - Valid JPEG (real JPEG magic bytes as base64) → 201
       - Valid PNG (real PNG magic bytes as base64) → 201

    2. **Invalid file types rejected** (3-4 tests):
       - EXE content disguised as .pdf → 400 with file validation error
       - ZIP file with .pdf extension → 400
       - Content type not in allowed list → 400

    3. **Size limit enforcement** (2 tests):
       - Content within limit → 201
       - Content exceeding limit → 400 (or 413 from body parser)

    4. **Filename security** (3-4 tests):
       - Normal filename accepted and stored correctly
       - Path traversal attempt (../../../etc/passwd) → sanitized or rejected
       - Filename with null bytes → sanitized
       - Filename with special characters → sanitized in response

    Use setupTestApp() and generateTestJWT() from test utilities. Mock Supabase storage and database calls. Create real binary buffers for magic number tests:

    - EXE: Buffer.from([0x4D, 0x5A, ...padding]) (MZ header)

    Convert to base64 for the request body content field.
