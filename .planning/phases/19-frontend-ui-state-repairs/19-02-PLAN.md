<task type="auto">
  <name>Task 2: Configure next.config.ts for backend API integration</name>
  <files>frontend/next.config.ts</files>
  <action>
Configure `next.config.ts` with:

1. **API rewrites** — Proxy `/api/v1/*` requests to the backend API URL to avoid CORS issues during development:
   ```typescript
   async rewrites() {
     return [
       {
         source: '/api/v1/:path*',
         destination: `${process.env.NEXT_PUBLIC_API_URL}/v1/:path*`,
       },
     ];
   }
   ```

2. **Security headers** — Add basic security headers matching what the backend already sets via Helmet:
   ```typescript
   async headers() {
     return [
       {
         source: '/(.*)',
         headers: [
           { key: 'X-Frame-Options', value: 'DENY' },
           { key: 'X-Content-Type-Options', value: 'nosniff' },
           { key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' },
         ],
       },
     ];
   }
   ```

3. **Image domains** — If the app loads images from external sources, configure `images.remotePatterns`. For now, leave empty since no external images are used.

Do NOT add experimental features or complex middleware — keep config minimal and focused on the bugs being fixed.
  </action>
  <verify>cd frontend && npm run build succeeds with the new config (rewrites and headers don't break build)</verify>
  <done>next.config.ts has API rewrites and security headers configured, build still passes</done>
