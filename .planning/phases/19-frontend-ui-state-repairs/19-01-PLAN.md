**Build error:** `unhandledRejection [TypeError: (0 , _lodashcurry.default) is not a function]`
**Root cause analysis:** Next.js 15.5.12 bundles `lodash.curry` at `next/dist/compiled/lodash.curry` which exports a CJS function. The ESM interop wrapper `_interop_require_default` wraps it as `{ default: fn }`, which works correctly in isolation. The crash occurs during the webpack/turbopack CSS processing pipeline, likely due to a PostCSS version conflict: `@tailwindcss/postcss@4.1.17` pulls `postcss@8.5.6` while `next@15.5.12` pins `postcss@8.4.31`. Dual PostCSS versions can corrupt the build pipeline.

**Current dependencies:**
- next@15.5.12
- @tailwindcss/postcss@4.1.17 (Tailwind v4 PostCSS plugin)
- tailwindcss@4.1.17
- postcss@8.5.6 (via @tailwindcss/postcss) AND postcss@8.4.31 (via next)
- Node.js v22.21.0

**Key files:**
@frontend/package.json
@frontend/postcss.config.mjs
@frontend/src/app/globals.css
@frontend/next.config.ts
