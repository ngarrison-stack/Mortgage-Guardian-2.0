<task type="auto">
  <name>Task 1: Create frontend CI workflow</name>
  <files>.github/workflows/frontend-ci.yml</files>
  <action>
1. Create `.github/workflows/frontend-ci.yml` with:
   - Name: "Frontend CI"
   - Triggers: push to main, pull_request to main
   - Concurrency: cancel-in-progress per branch (separate group from backend)
   - Single job "build" on ubuntu-latest:
     - actions/checkout@v4
     - actions/setup-node@v4 with node-version: '20', cache: 'npm', cache-dependency-path: 'frontend/package-lock.json'
     - `npm ci` in frontend/
     - `npm run lint` in frontend/ (uses next lint / eslint)
     - `npm run build` in frontend/
   - Environment variables for build:
     - NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY: "pk_test_placeholder" (Clerk SDK needs this at build time even if not functional — without it, build may fail with "missing env" error)
     - Set any other NEXT_PUBLIC_ vars that would cause build failure to placeholder values
     - Add comment explaining these are CI-only placeholders, not real credentials

2. Do NOT add: test step (no test suite exists), deployment steps, or type-check step (next build already runs tsc).

3. Check if `frontend/package-lock.json` exists. If not, the cache-dependency-path needs adjustment — use the lockfile that exists.
  </action>
  <verify>`python3 -c "import yaml; yaml.safe_load(open('.github/workflows/frontend-ci.yml'))"` parses without error</verify>
