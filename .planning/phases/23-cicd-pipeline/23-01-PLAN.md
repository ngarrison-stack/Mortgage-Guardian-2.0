<task type="auto">
  <name>Task 2: Add ESLint configuration to backend-express</name>
  <files>backend-express/.eslintrc.json, backend-express/package.json</files>
  <action>
1. Install ESLint as dev dependency: Add `eslint` to devDependencies in package.json (check current version compatibility with Node 20 — use ESLint 8.x, NOT 9.x which requires flat config and would be a bigger migration).

2. Create `backend-express/.eslintrc.json` with:
   - env: node, es2024, jest
   - extends: "eslint:recommended"
   - parserOptions: ecmaVersion 2024, sourceType commonjs (backend is CJS, not ESM)
   - rules: Keep minimal — no-unused-vars as warn (not error, to avoid blocking CI on existing code), no-console off (Winston handles logging, but some files may still have console)
   - ignorePatterns: node_modules, coverage, dist, __tests__/mocks

3. Add "lint" script to package.json scripts: `"lint": "eslint services/ routes/ middleware/ schemas/ utils/ server.js"`
   Target only source directories (not __tests__, not node_modules).

4. Run `cd backend-express && npx eslint services/ routes/ middleware/ schemas/ utils/ server.js --max-warnings=50` to verify it doesn't produce blocking errors. If there are errors (not warnings), fix the ESLint config rules to be lenient enough that CI passes — the goal is a green CI, not a lint cleanup (that's a separate concern). Use `--max-warnings` flag in the lint script if needed.

Do NOT use ESLint 9 flat config. Do NOT add TypeScript ESLint parser (backend is plain JS). Do NOT add Prettier (out of scope).
  </action>
  <verify>`cd backend-express && npm run lint` exits 0 (or exits 0 with warnings only)</verify>
  <done>ESLint config exists, lint script runs without errors, ready for CI</done>
</task>
