<task type="auto">
  <name>Task 1: Install autocannon and create load test runner</name>
  <files>backend-express/package.json, backend-express/loadtest/runner.js, backend-express/loadtest/config.js</files>
  <action>
Install autocannon as a dev dependency (`npm install -D autocannon`). Create `backend-express/loadtest/` directory with:

1. `config.js` — shared configuration:
   - BASE_URL defaulting to `http://localhost:3000`
   - DEFAULT_DURATION: 10 (seconds)
   - DEFAULT_CONNECTIONS: 10
   - Export a `runTest(opts)` function that wraps autocannon programmatic API, accepts { url, method, duration, connections, headers, body } and returns the result object

2. `runner.js` — CLI entry point:
   - Accept `--suite` argument (health, api, stress, all)
   - Dynamically import and run the matching suite file
   - Print summary table: latency (p50/p95/p99), req/sec, errors, timeouts
   - Exit code 1 if any p99 > 1000ms (1s target) or error rate > 1%

3. Add npm script: `"loadtest": "node loadtest/runner.js"` and `"loadtest:health": "node loadtest/runner.js --suite health"`

Do NOT use TypeScript — load tests are plain JS tooling scripts. Keep it simple, no frameworks beyond autocannon.
  </action>
  <verify>cd backend-express && npm ls autocannon && node loadtest/runner.js --help</verify>
  <done>autocannon in devDependencies, runner.js prints usage, config.js exports runTest function</done>
</task>

<task type="auto">
  <name>Task 2: Health endpoint baselines</name>
  <files>backend-express/loadtest/suites/health.js</files>
  <action>
Create `backend-express/loadtest/suites/health.js` that tests three endpoints:

1. **GET /health/live** — liveness probe (should be fastest, no external checks)
   - 10 connections, 10 seconds
   - Expected: p99 < 50ms, > 1000 req/sec

2. **GET /health/ready** — readiness probe (checks Supabase/Redis)
   - 10 connections, 10 seconds
   - Note: will be slower due to external checks, may fail if no Supabase/Redis

3. **GET /metrics** — metrics snapshot
   - 10 connections, 10 seconds
   - Expected: p99 < 100ms (in-memory computation)

Each test: use the runTest() from config.js, collect results, print formatted table with:
- Endpoint, Requests, Req/Sec, Latency p50/p95/p99, Errors, Timeouts

Handle the case where server is not running: print clear error message "Server not running at {BASE_URL}. Start with: npm run dev".

Do NOT start the server from the test script — user starts it separately. This is intentional to allow testing against any environment (local, staging, production).
  </action>
  <verify>cd backend-express && npm run dev & sleep 2 && node loadtest/runner.js --suite health; kill %1</verify>
  <done>Health suite runs against local server, prints table with p50/p95/p99 latencies, req/sec for all 3 health endpoints</done>
</task>
